# 备件管理系统 - 系统管理模块功能扩展开发文档

## 一、系统管理模块业务分析

### 1.1 当前系统管理模块功能

#### 已有功能
1. **系统概览页面** - 显示系统统计数据和快速操作入口
2. **邮件配置管理** - SMTP 服务器配置和测试
3. **系统参数设置** - 库存预警规则和通知方式
4. **系统日志记录** - 自动记录用户操作日志

#### 数据模型
- `EmailConfig` - 邮件配置
- `UserEmailConfig` - 用户邮箱配置
- `Notification` - 系统通知
- `Alert` - 告警信息
- `AlertRule` - 告警规则
- `SystemLog` - 系统日志
- `Tag` - 标签管理

### 1.2 备件管理系统业务特点

#### 核心业务需求
1. **库存管理复杂性** - 需要多级库存状态监控和预警
2. **用户角色多样性** - 管理员、仓库管理员、采购员、维修管理员、财务人员等
3. **数据安全性要求高** - 涉及企业资产和采购信息
4. **操作流程规范性** - 需要严格的审批流程和审计追踪
5. **实时性要求** - 库存变化需要及时通知相关人员
6. **报表统计需求** - 需要多维度数据分析

#### 系统管理痛点
1. 缺乏系统运行状态监控
2. 没有数据备份和恢复管理界面
3. 缺少操作审计和日志分析
4. 权限管理不够精细化
5. 缺乏系统性能监控
6. 没有字典管理，数据标准化程度低
7. 缺少定时任务管理
8. 缺乏 API 接口管理
9. 没有系统参数集中配置
10. 缺少帮助文档和使用指南

#### 功能描述
实时监控系统的运行状态，包括服务器资源、数据库状态、缓存状态、服务健康检查等。

#### 功能点
1. **服务器状态监控**
   - CPU 使用率
   - 内存使用率
   - 磁盘空间
   - 系统运行时间

2. **数据库监控**
   - 连接状态
   - 连接池使用情况
   - 慢查询监控
   - 数据库大小

3. **缓存服务监控**
   - Redis 连接状态
   - 缓存命中率
   - 缓存键数量
   - 内存使用情况

4. **服务健康检查**
   - 邮件服务状态
   - 各模块可用性
   - 外部接口连通性

#### 数据模型
```python
class SystemMonitor(db.Model):
    __tablename__ = 'system_monitor'
    
    id = db.Column(db.Integer, primary_key=True)

    metric_name = db.Column(db.String(100), nullable=False, comment='指标名称')
    metric_value = db.Column(db.Float, nullable=False, comment='指标值')
    threshold_warning = db.Column(db.Float, comment='警告阈值')

    status = db.Column(db.String(20), default='normal', comment='状态')
    recorded_at = db.Column(db.DateTime, default=datetime.now, comment='记录时间')
```

        return decorated_function
    return decorator
