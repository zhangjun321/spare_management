# 百度文心一格仓库图像生成 - 开发完成报告

## 一、功能概述

成功实现基于百度文心一格的仓库实景图 AI 生成功能，可将仓库布局数据自动转换为高质量的视觉图像。

### 核心能力

1. **数据驱动的图像生成** - 基于真实仓库布局数据生成图像
2. **智能提示词转换** - 自动将结构化数据转换为 AI 提示词
3. **多视角生成** - 支持入口、中心、货架、通道、俯视等多种视角
4. **多风格支持** - 写实、艺术、动漫等多种风格
5. **实时动态更新** - 仓库数据变化时重新生成图像

### ✅ 3. AI 图像生成 API

**文件**: `app/routes/ai_image_routes.py`

**API 接口**:

1. **单张生成** - `POST /ai-image/generate/<warehouse_id>/`
   - 参数：style, focus, resolution
   - 返回：生成的图像（base64 或 URL）

2. **批量生成** - `POST /ai-image/generate-batch/`
   - 一次性生成 5 个不同视角的图像
   - 返回：图像数组

3. **自定义提示词** - `POST /ai-image/update-prompt/`
   - 允许手动调整提示词
   - 重新生成图像

4. **生成历史** - `GET /ai-image/history/<warehouse_id>/`
   - 查询历史生成记录

5. **API 测试** - `GET /ai-image/api/test/`
   - 测试百度 API 连接状态

### ✅ 4. 实时动态更新机制

**实现方式**:
1. 监听仓库数据变化（库区、货架、备件等）
2. 检测到变化后自动重新生成提示词
3. 调用百度 API 生成新图像
4. 更新图像缓存

**触发条件**:
- 仓库布局变更
- 库区配置调整
- 货架数量/类型变化
- 备件信息更新
- 库存状态重大变化

**更新策略**:
- 实时生成：用户手动触发时立即生成
- 延迟更新：数据变化后延迟 5 分钟生成（避免频繁调用）
- 批量更新：夜间闲时批量更新所有仓库图像

### 3.2 视角变体生成

**5 种预设视角**:
1. **entrance** - 从仓库入口处视角，展现整体布局
2. **center** - 从仓库中心视角，360 度全景展示
3. **shelves** - 近距离拍摄货架区域，展示备件存储细节
4. **path** - 沿着主通道视角，展示通道和两侧货架
5. **aerial** - 俯视角度，展示仓库整体规划布局

### 3.3 API 调用流程

```
1. 获取仓库 ID
   ↓
2. 查询仓库布局数据
   ↓
3. 转换为提示词
   ↓
4. 获取百度 API 访问令牌
   ↓
5. 调用文心一格 API
   ↓
6. 接收生成的图像
   ↓
7. 返回给前端展示
```

## 四、使用指南

### 4.1 基本使用流程

1. **访问页面**
   - 侧边栏 → AI 图像生成
   - URL: `/ai-image/`

2. **选择仓库**
   - 从下拉列表选择目标仓库

3. **配置参数**
   - 选择图像风格（写实/艺术/动漫）
   - 选择视角焦点（入口/中心/货架等）
   - 选择分辨率（512x512/1024x768/1024x1024）

4. **生成图像**
   - 点击"生成图像"按钮
   - 等待 30-60 秒
   - 查看生成结果

5. **保存图像**
   - 点击下载按钮保存图像
   - 或复制提示词用于其他用途

### 4.2 高级功能

**批量生成**:
- 点击"批量生成"按钮
- 一次性生成 5 个不同视角的图像
- 适合全面了解仓库布局

**自定义提示词**:
- 在提示词文本框手动编辑
- 点击"重新生成"应用自定义提示词
- 获得更符合特定需求的图像

**API 测试**:
- 点击"测试 API 连接"按钮
- 验证百度 API 配置是否正确
- 快速排查连接问题

## 七、优势与局限性

### ✅ 优势

1. **自动化程度高** - 无需手动编写提示词
2. **专业性强** - 针对仓库场景优化
3. **灵活性好** - 支持多种风格和视角
4. **集成度高** - 与现有系统无缝对接
5. **可扩展性强** - 易于添加新功能

### ⚠️ 局限性

1. **依赖外部 API** - 需要百度 API 凭证和网络连接
2. **生成时间较长** - 单次生成需 30-60 秒
3. **成本考虑** - API 调用可能产生费用
4. **实时性限制** - 不适合秒级实时更新场景
5. **一致性挑战** - 多次生成可能存在风格差异
