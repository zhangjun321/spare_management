# 设备管理模块增强说明

## 概述

本次设备管理模块增强基于当前行业标准和发展趋势，融入了工业4.0、预测性维护、IoT数据采集等现代设备管理理念。保持了与现有MySQL数据库架构的完全兼容。

## 新增功能

### 1. 设备健康指数管理

**功能描述**：
- 综合健康评分（0-100分）
- 健康等级划分（优秀/良好/正常/警告/严重）
- 可靠性评分、性能评分、维护状态评分
- 风险等级评估

**数据模型**：`EquipmentHealthIndex`

**API接口**：
- `GET /api/equipment/{id}/health/current` - 获取当前健康状态
- `GET /api/equipment/{id}/health/history` - 获取健康历史趋势

### 2. IoT数据采集与监控

**功能描述**：
- 实时设备运行状态监控
- 温度、振动、电压、电流等关键参数采集
- 射线源功率、照射时间监控
- 今日检测次数、累计检测次数统计
- 数据质量标记

**数据模型**：`EquipmentIotData`

**API接口**：
- `GET /api/equipment/{id}/iot/latest` - 获取最新IoT数据
- `GET /api/equipment/{id}/iot/history` - 获取IoT历史趋势
- `POST /api/equipment/{id}/iot/submit` - 提交IoT数据

### 3. 预测性维护分析

**功能描述**：
- 故障预测（MTBF预测、剩余使用寿命预测）
- 故障风险概率计算
- 预测置信度评估
- 维护建议自动生成
- 预测模型版本管理

**数据模型**：`EquipmentPredictiveMaintenance`

**API接口**：
- `GET /api/equipment/{id}/predictive/analysis` - 获取预测分析
- `POST /api/equipment/{id}/predictive/acknowledge/{prediction_id}` - 确认预测通知

### 4. 设备性能指标（OEE）

**功能描述**：
- 整体设备效率（OEE）计算
- 可用率（Availability）
- 性能率（Performance）
- 合格率（Quality）
- 计划运行时间、实际运行时间、停机时间、维护时间统计
- 检测统计（成功/失败/拒检次数）
- 质量指标（缺陷率、误报率）
- 能效指标（能耗、能效评分）
- 成本指标（维护成本、运营成本、总成本）

**数据模型**：`EquipmentPerformanceMetric`

**API接口**：
- `GET /api/equipment/{id}/performance/dashboard` - 性能仪表板数据
- `GET /api/equipment/{id}/performance/history` - 性能历史趋势

### 5. 设备组件与备件管理

**功能描述**：
- 设备关键组件台账
- 组件类型（关键/重要/标准）
- 安装位置、安装日期记录
- 更换周期、预期寿命设置
- 已使用时间统计
- 状态监控（正常/警告/需更换）
- 备件关联管理

**数据模型**：`EquipmentComponent`

**API接口**：
- `GET /api/equipment/{id}/components` - 获取设备组件列表

### 6. 维护任务管理

**功能描述**：
- 任务类型（预防性/纠正性/点检/校准/升级）
- 优先级设置（低/中/高/严重）
- 计划时间、预估工时管理
- 任务进度跟踪（0-100%）
- 任务状态流转（待处理/进行中/已完成/已取消/逾期）
- 检查清单管理
- 所需工具、备件关联
- 执行结果记录
- 成本统计（材料成本/人工成本/总成本）

**数据模型**：`EquipmentMaintenanceTask`

**API接口**：
- `GET /api/equipment/{id}/tasks` - 获取维护任务列表
- `POST /api/equipment/{id}/tasks/create` - 创建维护任务
- `PUT /api/equipment/{id}/tasks/{task_id}/update` - 更新维护任务

### 7. 设备文档与知识库管理

**功能描述**：
- 文档类型（操作手册/图纸/规格/维护规程/维修记录）
- 文档版本管理
- 分类标签系统
- 公开/私有权限控制
- 审核流程（草稿/已审核/已批准）
- 下载统计
- 文件元数据管理

**数据模型**：`EquipmentDocument`

**API接口**：
- `GET /api/equipment/{id}/documents` - 获取设备文档列表

## 注意事项

1. **数据库迁移**：新增模型表需要数据库迁移（建议使用Flask-Migrate）
2. **性能优化**：IoT数据量大时建议分表或归档策略
3. **安全性**：IoT数据提交接口建议添加认证机制
4. **浏览器兼容性**：Chart.js需要现代浏览器支持
