# 数据备份管理问题修复总结

## 问题根因分析

经过全面的代码审查和调试，发现以下关键问题：

### 1. **错误处理器识别 API 请求的逻辑错误** ❌

**问题位置**: `app/errors/handlers.py` 第 12 行

**原代码**:
```python
def is_api_request():
    """判断是否是 API 请求"""
    return request.path.startswith('/api/') or 'application/json' in request.accept_mimetypes
```

**问题分析**:
- 备份 API 的路径是 `/backup/api/create`，不是以 `/api/` 开头
- 导致所有 `/backup/api/*` 的错误都被当作普通页面请求处理
- 返回 HTML 错误页面而不是 JSON，前端解析失败

**修复方案**:
```python
def is_api_request():
    """判断是否是 API 请求"""
    # 检查路径是否包含 /api/ 或者 Accept 头是否包含 application/json
    return '/api/' in request.path or 'application/json' in request.accept_mimetypes
```

### 2. **CSP (内容安全策略) 配置问题** ⚠️

**问题位置**: `app/__init__.py` 第 88 行

**原代码**:
```python
response.headers['Content-Security-Policy'] = "... connect-src 'self' http://localhost:* http://127.0.0.1:* https://*"
```

**问题分析**:
- CSP 的 `connect-src` 指令限制了 AJAX/Fetch 请求的目标
- 某些浏览器可能严格执行此策略

**修复方案**:
```python
response.headers['Content-Security-Policy'] = "... connect-src 'self' http://localhost:* http://127.0.0.1:* https://* 'unsafe-inline'"
```

### 3. **JSON 响应编码问题** ⚠️

**问题位置**: `app/routes/backup.py` 第 40-105 行

**问题分析**:
- 使用 `jsonify()` 返回的中文可能出现乱码
- 需要显式设置响应头和编码

**修复方案**:
```python
# 直接返回 JSON 字符串
json_str = json.dumps(response_dict, ensure_ascii=False)
from flask import make_response
response = make_response(json_str)
response.headers['Content-Type'] = 'application/json; charset=utf-8'
return response, status_code
```
