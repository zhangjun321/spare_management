# 设备管理模块增强功能说明

## 📋 概述

本次更新为设备管理模块添加了多项增强功能，包括状态变更管理、站点名称、仪表盘优化等。

## 🎯 新增功能

### 1. 状态变更管理
- **快速变更**：在设备列表和详情页都提供了状态变更按钮
- **历史记录**：所有状态变更都会被记录，包括变更原因、时间、操作人
- **状态选项**：
  - 运行中 (running)
  - 停机 (stopped)
  - 维修中 (maintenance)
  - 已报废 (scrapped)

### 2. 站点名称
- **自动生成**：格式为 `{系列}-{城市}-{序号}`，例如 `FS-北京-001`
- **筛选功能**：在仪表盘中支持按站点名称筛选
- **列表展示**：在设备列表页显示站点名称

### 3. 仪表盘增强
- **新增统计卡片**：
  - 保修即将到期（30天内）
  - 已过保修
- **优化筛选**：
  - 站点名称筛选
  - 设备类型筛选
- **信息窗口**：显示站点名称信息

### 4. 设备详情页优化
- **状态变更按钮**：直接在详情页变更状态
- **状态变更历史**：查看所有历史变更记录
- **更完整的信息**：展示站点名称

## 🗄️ 数据库变更

### 新增字段
- `equipment.site_name`: 站点名称（VARCHAR(100)）

### 新增表
- `equipment_status_history`: 状态变更历史记录表

## 🚀 快速开始

### 1. 数据库迁移
运行迁移脚本添加新字段和生成站点名称：
```bash
python scripts/migrate_equipment.py
```

### 2. 访问功能
- **设备仪表盘**：`/equipment/dashboard`
- **设备列表**：`/equipment/`
- **设备详情**：`/equipment/<id>`

## 📊 API接口

### 概览统计
```
GET /equipment/api/overview-stats
```
返回：设备总数、各状态数量、保修预警数量等

### 位置数据（含筛选）
```
GET /equipment/api/locations?site_name=XX&series=XX
```

### 状态变更历史
```
GET /equipment/<id>/status-history
```

### 变更状态
```
POST /equipment/<id>/change-status
```
参数：
- `new_status`: 新状态
- `change_reason`: 变更原因

## 🎨 UI特性

### 状态颜色
- 🟢 运行中
- ⚪ 停机
- 🟠 维修中
- 🔴 已报废

### 统计卡片渐变
- 运行中：绿色渐变
- 维修中：橙色渐变
- 保修即将到期：红色渐变
- 已过保修：紫色渐变

## 📝 使用说明

### 变更设备状态
1. 在设备列表或详情页点击"变更状态"按钮
2. 选择新状态
3. 输入变更原因（可选）
4. 提交后状态立即更新并记录

### 站点名称格式
站点名称格式：`{系列}-{城市}-{序号}`
- **系列**：FS/MT/CS/MB/FG/PB/RF
- **城市**：从设备地址中自动提取
- **序号**：设备ID取模，3位数字

### 仪表盘筛选
在仪表盘顶部的筛选器中：
- 选择站点名称可查看特定站点的设备
- 选择设备类型可查看特定系列的设备
- 点击状态图例可切换该状态设备的显示/隐藏

## 🔧 扩展模块（预留）

已预留的扩展模块模型：
- `EquipmentChecklist`: 设备点检表
- `EquipmentAttachment`: 设备附件
- `EquipmentTag`: 设备标签
- `EquipmentMaintenancePlan`: 保养计划

这些模块可在后续开发中启用。

## 📌 注意事项

1. 状态变更会记录到历史，建议填写变更原因
2. 站点名称是自动生成的，也可以手动修改
3. 保修到期预警基于`warranty_expiry`字段计算
4. 数据库迁移脚本只添加新字段，不破坏现有数据

## 🎉 总结

本次升级为设备管理模块提供了更完善的状态管理、更好的可视化和更便捷的筛选功能。
