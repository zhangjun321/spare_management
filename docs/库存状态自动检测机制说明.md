### 3. 库存状态判断逻辑

**文件**：`app/models/spare_part.py`

```python
def update_stock_status(self):
    """更新库存状态"""
    if self.current_stock == 0:
        self.stock_status = 'out'  # 缺货
    elif self.current_stock <= self.min_stock:
        self.stock_status = 'low'  # 低库存
    elif self.max_stock and self.current_stock >= self.max_stock:
        self.stock_status = 'overstocked'  # 超储
    else:
        self.stock_status = 'normal'  # 正常
```

**判断规则**：

| 条件 | 状态 | 说明 |
|------|------|------|
| `current_stock = 0` | out（缺货） | 库存为 0 |
| `0 < current_stock ≤ min_stock` | low（低库存） | 低于最低库存 |
| `min_stock < current_stock < max_stock` | normal（正常） | 合理范围 |
| `current_stock ≥ max_stock` | overstocked（超储） | 超过最高库存 |

## ⚠️ 注意事项

### 1. 批量导入数据时

如果使用脚本批量导入数据，**必须**在导入后调用 `update_stock_status()` 或执行 SQL 更新：

**方法 1：Python 脚本**
```python
for part_data in parts_data:
    spare_part = SparePart(**part_data)
    spare_part.update_stock_status()  # ✅ 必须调用
    db.session.add(spare_part)

db.session.commit()
```

**方法 2：SQL 更新**
```sql
UPDATE spare_part 
SET stock_status = CASE
    WHEN current_stock = 0 THEN 'out'
    WHEN current_stock <= min_stock THEN 'low'
    WHEN max_stock IS NOT NULL AND current_stock >= max_stock THEN 'overstocked'
    ELSE 'normal'
END;
```

---

### 2. 批量更新库存时

如果直接更新数据库（如 Excel 导入），需要**手动刷新**库存状态：

```bash
python scripts/refresh_stock_status.py
```

---

### 3. 修改库存阈值时

如果修改了 `min_stock` 或 `max_stock`，库存状态**不会自动更新**，需要：

**方法 1**：重新保存备件（会调用 `update_stock_status()`）

**方法 2**：运行刷新脚本
```bash
python scripts/refresh_stock_status.py
```
