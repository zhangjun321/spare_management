# 仓库管理功能访问指南

## ✅ 仓库列表确实存在！

系统目前有**两套仓库管理系统**并行，导致访问路径混淆。

## 📦 方案一：Flask 传统版本（推荐访问）

### 访问地址
```
http://localhost:5000/warehouses/
```

### 功能特点
- ✅ **仓库列表** - 表格形式展示
- ✅ **筛选搜索** - 按名称/编码/类型/状态筛选
- ✅ **分页功能** - 20 条/页
- ✅ **统计卡片** - 总仓库数/启用数/禁用数
- ✅ **快速操作** - 创建/编辑/删除
- ✅ **快速创建示例** - 一键创建 6 个示例仓库

### 页面路由
| 功能 | URL | 说明 |
|------|-----|------|
| 仓库列表 | `/warehouses/` | 主页面 |
| 创建仓库 | `/warehouses/create` | 新建表单 |
| 仓库详情 | `/warehouses/<id>` | 查看详情 |
| 编辑仓库 | `/warehouses/<id>/edit` | 编辑表单 |
| 库位列表 | `/warehouses/locations/` | 库位管理 |
| 库存概览 | `/warehouses/inventory/` | 库存查询 |
| 操作记录 | `/warehouses/operations/` | 历史记录 |
| 报表统计 | `/warehouses/reports/*` | 各类报表 |

### 特色功能
- **快速创建示例仓库**
  ```
  http://localhost:5000/warehouses/quick-create-warehouses/
  ```
  一键创建 6 个示例仓库（主仓库、冷藏仓库、危险品仓库等）

## 🔍 问题分析

### 为什么会有两个版本？

这是历史原因造成的：
1. **Flask 版本** - 原始版本，使用 Flask 模板渲染
2. **React 版本** - 新版本，使用前后端分离架构

### 当前问题

**菜单链接混乱** - 系统中可能存在指向不同版本的菜单链接

**代码冗余** - 两套代码实现相同功能
