### 2. AI 服务开发

#### 2.1 AI 货位推荐服务（[`app/services/ai_location_service.py`](file:///d:/Trae/spare_management/app/services/ai_location_service.py)）

**核心功能**：
- `recommend_warehouse_and_location(spare_part_data)` - AI 智能推荐仓库和货位

**AI 考虑因素**：
1. 仓库类型匹配（危险品、冷库等特殊要求）
2. 仓库专长（该仓库常存储的备件类型）
3. 货位尺寸和承重（满足备件存储要求）
4. 出入库频率（高频备件靠近出口）
5. 价值等级（高价值备件放在安全位置）
6. 仓库利用率（避免过度拥挤的仓库）
7. 存储要求（防潮、防尘等）

**返回结果**：
```json
{
  "success": true,
  "recommended_warehouse_id": 1,
  "recommended_warehouse_name": "北京一号仓库",
  "recommended_location_id": 10,
  "recommended_location_code": "A-01-02-03",
  "confidence": 92,
  "reason": "该仓库专门存储轴承类备件，货位靠近出口，便于高频拣货",
  "alternative_warehouses": [2, 3],
  "notes": "建议定期检查润滑状态"
}
```

## 🎯 混合模式实现

### 人工录入 + AI 推荐

系统实现了**人工录入与 AI 推荐相结合**的混合模式：

#### 人工录入方式
1. **手动选择**：用户在表单中通过下拉框选择仓库和货位
2. **批量导入**：通过 Excel 导入，使用仓库编码和货位编码自动关联
3. **完全控制**：用户可以忽略 AI 推荐，完全手动选择

#### AI 推荐方式
1. **智能分析**：AI 分析备件特性（名称、规格、分类、尺寸、重量等）
2. **自动推荐**：返回最佳仓库和货位推荐
3. **置信度**：提供推荐置信度（0-100）
4. **推荐理由**：详细说明推荐原因

#### 混合模式工作流程
```
1. 用户填写备件基本信息
   ↓
2. 点击"AI 推荐"按钮（可选）
   ↓
3. AI 返回推荐仓库和货位
   ↓
4. 用户选择：
   - ✅ 接受推荐 → 自动填入表单
   - ✏️ 修改推荐 → 手动调整
   - ❌ 忽略推荐 → 完全手动选择
   ↓
5. 保存备件信息
```

## 🔧 技术难点与解决方案

### 1. 模型冲突问题

**问题**：系统中已存在 `InventoryRecord`、`WarehouseZone` 等模型，重复定义导致冲突。

**解决方案**：
- 使用已有的模型文件
- 只在需要的地方添加关系引用
- 避免重复定义相同的模型类

### 2. MySQL 保留字冲突

**问题**：`column` 是 MySQL 保留字，直接使用会导致 SQL 语法错误。

**解决方案**：
- 使用 `column_val` 替代 `column`
- 或者在 SQL 中使用反引号 `` `column` ``

### 3. MySQL 索引创建语法

**问题**：MySQL 不支持 `CREATE INDEX IF NOT EXISTS` 语法。

**解决方案**：
- 先查询现有索引
- 如果索引不存在再创建
- 使用 `inspector.get_indexes()` 检查

### 4. Windows 控制台编码

**问题**：emoji 字符在 Windows 控制台 GBK 编码下无法显示。

**解决方案**：
- 移除所有 emoji 字符
- 使用 `[OK]`、`[-]`、`[ERROR]` 等文本标识
