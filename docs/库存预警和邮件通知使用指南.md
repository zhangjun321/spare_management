# 库存预警和邮件通知系统使用指南

## 功能概述

系统现已实现完整的库存预警和邮件通知功能，包括：

1. **自动库存预警**：当备件库存状态变化时自动触发预警
2. **系统内通知**：在系统右上角显示通知消息
3. **邮件通知**：自动发送邮件给所有系统管理员
4. **邮件配置管理**：提供专门的邮件配置页面

## 库存预警逻辑

### 预警触发条件

系统根据以下规则自动判断库存状态：

| 库存状态 | 触发条件 | 预警级别 |
|---------|---------|---------|
| **缺货** | 当前库存 = 0 | 危险 (danger) |
| **低库存** | 0 < 当前库存 ≤ 最低库存 | 警告 (warning) |
| **正常** | 最低库存 < 当前库存 < 最高库存 | 无预警 |
| **超储** | 当前库存 ≥ 最高库存 | 提示 (info) |

### 预警触发时机

预警在以下情况自动触发：
- 创建新备件时，库存状态不为"正常"
- 编辑备件时，库存状态发生变化且变为非"正常"状态

## 配置步骤

### 1. 配置邮件服务器

访问路径：**系统管理 → 邮件配置** (`/system/email-config`)

#### 常用邮箱 SMTP 配置

| 邮箱服务商 | SMTP 服务器 | 端口 | 说明 |
|-----------|------------|------|------|
| QQ 邮箱 | smtp.qq.com | 465/587 | 需使用授权码 |
| 163 邮箱 | smtp.163.com | 465/587 | 需使用授权码 |
| Gmail | smtp.gmail.com | 465/587 | 需开启 SMTP 服务 |
| Outlook | smtp.office365.com | 587 | 需使用 TLS |

#### 配置步骤

1. 进入"邮件配置"页面
2. 填写 SMTP 服务器信息
3. 输入用户名和密码（或授权码）
4. 设置发件人邮箱和名称
5. 选择是否使用 SSL/TLS 加密
6. 点击"测试连接"验证配置
7. 保存配置

### 2. 配置系统参数

访问路径：**系统管理 → 系统参数** (`/system/settings`)

#### 可配置项

- **启用库存预警**：总开关，控制是否自动检测库存
- **低库存预警**：是否对低库存状态触发预警
- **缺货预警**：是否对缺货状态触发预警
- **超储预警**：是否对超储状态触发预警
- **邮件通知**：是否发送邮件通知
- **系统内通知**：是否在系统内显示通知消息

### 3. 确保管理员有邮箱地址

访问路径：**系统管理 → 用户管理**

确保所有系统管理员账户都设置了有效的邮箱地址，否则无法接收邮件通知。

## 通知接收

### 系统内通知

1. 点击右上角的铃铛图标查看通知
2. 未读通知会显示红色标记和"新"标签
3. 点击通知可跳转到详情页
4. 支持标记为已读、删除等操作
5. 访问 `/notification` 查看所有通知

### 邮件通知

当库存预警触发时，系统会自动向所有管理员发送邮件：

**邮件内容包含：**
- 备件代码和名称
- 规格型号
- 当前库存状态
- 当前库存、最低库存、最高库存
- 存放位置
- 预警时间

## 查看和管理预警

### 仪表盘预警

在仪表盘首页显示：
- 库存预警卡片（显示最近 5 个预警）
- 点击可跳转到备件详情页

### 通知中心

访问 `/notification` 查看：
- 所有通知列表
- 按类型筛选（全部、库存预警）
- 标记为已读/删除通知
- 批量标记已读

### 告警记录

系统会保存所有预警记录在 `alert` 表中，可通过数据库查询历史记录。

## 测试功能

### 测试邮件连接

在"邮件配置"页面点击"测试连接"按钮，系统会尝试连接 SMTP 服务器并返回结果。

### 手动触发预警测试

1. 选择一个正常状态的备件
2. 编辑该备件，修改当前库存：
   - 设为 0（触发缺货预警）
   - 设为低于最低库存（触发低库存预警）
   - 设为高于最高库存（触发超储预警）
3. 保存后系统会自动：
   - 更新库存状态
   - 创建预警记录
   - 创建系统内通知
   - 发送邮件给所有管理员

## 技术实现

```
    │   ├── index.html
    │   ├── email_config.html
    │   └── settings.html
    └── notification/          # 通知管理模板
        └── list.html
```

### 核心代码

**预警触发流程：**

1. `SparePart.update_stock_status()` - 检测库存状态变化
2. `SparePart._trigger_alert()` - 触发预警
3. 创建 `Alert` 记录
4. 为每个管理员创建 `Notification`
5. 调用 `EmailService.send_stock_alert()` 发送邮件

### 自动刷新

系统每 30 秒自动刷新一次通知，确保及时显示最新预警。

## 常见问题

### Q: 为什么没有收到邮件？

A: 检查以下几点：
1. 邮件服务器配置是否正确
2. 管理员账户是否设置了邮箱
3. 检查垃圾邮件文件夹
4. 查看系统日志是否有发送失败的记录

### Q: 如何关闭邮件通知？

A: 在"系统参数"中关闭"邮件通知"选项

### Q: 预警记录保存多久？

A: 预警记录永久保存在数据库中，可手动清理

### Q: 可以自定义预警阈值吗？

A: 当前使用系统默认阈值（基于 min_stock 和 max_stock），后续版本将支持自定义规则

## 日志查看

系统会将所有邮件发送日志记录到 `logs/` 目录下，包含：
- 发送成功记录
- 发送失败原因
- 连接错误信息

可通过查看日志文件排查问题。
