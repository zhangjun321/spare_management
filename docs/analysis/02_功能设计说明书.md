## 二、新增功能详细设计

### F001：采购管理模块

#### 2.1.1 功能概述
利用现有 `PurchasePlan`、`PurchaseRequest`、`PurchaseOrder`、`PurchaseOrderItem`、`PurchaseQuote` 模型，提供完整的采购业务 UI。

#### 2.1.2 业务流程
```
库存触发预警
    │
    ▼
采购需求申请 (PurchaseRequest)
    │ status: pending
    ▼
审批 → 拒绝 (rejected)
    │ approved
    ▼
供应商报价 (PurchaseQuote)
    │ is_selected = true
    ▼
生成采购订单 (PurchaseOrder)
    │ status: pending→approved→received
    ▼
收货入库 → 自动创建 InboundOrder
    │
    ▼
完成 (completed)
```

#### 2.1.3 采购需求页面设计

**页面**：`/purchase/requests`

| 区域 | 内容 |
|------|------|
| 顶部筛选栏 | 备件名称/代码、状态（pending/approved/rejected）、日期范围 |
| 列表列 | 申请编号、备件名称、申请数量、预估单价、申请人、申请日期、状态、操作 |
| 操作按钮 | 新增申请、审批（管理员）、查看详情 |
| 低库存联动 | 页面加载时调用 `/api/spare-parts/?stock_status=low,out`，展示需关注备件列表，支持一键生成需求申请 |

#### 2.1.4 采购订单页面设计

**页面**：`/purchase/orders`

| 区域 | 内容 |
|------|------|
| 顶部 KPI 卡 | 本月采购金额、待收货订单数、超期订单数 |
| 列表列 | 订单编号、供应商、总金额、订单日期、预计交货日、实际交货日、状态 |
| 操作 | 新建订单、查看明细、确认收货 |

#### 2.1.5 收货页面设计

**页面**：`/purchase/receipts`

收货操作触发 `InboundOrder` 创建，字段映射：
```
PurchaseOrder.supplier_id → InboundOrder.supplier_id
PurchaseOrderItem.spare_part_id → InboundOrder.spare_part_id
PurchaseOrderItem.quantity → InboundOrder.quantity
inbound_type = 'purchase'
```

### F002：综合报表中心

#### 2.2.1 库存报表

**API**：`GET /api/reports/inventory`

**参数**：
```json
{
  "warehouse_id": 1,
  "category_id": 2,
  "date": "2026-04",
  "group_by": "category|supplier|warehouse"
}
```

**查询逻辑（伪代码）**：
```python
SELECT
  sp.category_id, c.name AS category_name,
  COUNT(DISTINCT sp.id) AS part_count,
  SUM(ir.quantity) AS total_quantity,
  SUM(ir.quantity * ir.unit_cost) AS total_value,
  SUM(CASE WHEN sp.stock_status='low' THEN 1 ELSE 0 END) AS low_stock_count,
  SUM(CASE WHEN sp.stock_status='out' THEN 1 ELSE 0 END) AS out_of_stock_count
FROM spare_part sp
JOIN inventory_record ir ON ir.spare_part_id = sp.id
LEFT JOIN category c ON c.id = sp.category_id
GROUP BY sp.category_id
```

**前端图表**：
- 饼图：按分类的库存价值占比
- 柱状图：各仓库库存数量对比
- 数据表：支持导出 Excel

#### 2.2.2 收发存汇总报表

**API**：`GET /api/reports/inbound-outbound`

**参数**：
```json
{
  "start_date": "2026-04-01",
  "end_date": "2026-04-14",
  "warehouse_id": 1,
  "spare_part_id": null
}
```

**报表列**：期初库存、本期入库、本期出库、期末库存、库存价值

#### 2.2.3 备件消耗分析报表

**API**：`GET /api/reports/consumption`

```sql
SELECT
  sp.part_code, sp.name,
  SUM(oo.quantity) AS total_consumed,
  COUNT(oo.id) AS order_count,
  AVG(oo.unit_cost) AS avg_cost,
  SUM(oo.total_amount) AS total_cost
FROM outbound_order oo
JOIN spare_part sp ON sp.id = oo.spare_part_id
WHERE oo.status = 'completed'
  AND oo.completed_at BETWEEN :start_date AND :end_date
GROUP BY oo.spare_part_id
ORDER BY total_consumed DESC
LIMIT 50
```

**前端图表**：
- 条形图：TOP 20 消耗备件
- 折线图：月度消耗趋势（按分类）

### F003：库存调拨模块

#### 2.3.1 数据模型

新增 `TransferOrder` 表：
```python
class TransferOrder(db.Model):
    __tablename__ = 'transfer_order'
    
    id = db.Column(db.Integer, primary_key=True)
    order_no = db.Column(db.String(50), unique=True, nullable=False)  # TF20260414001
    
    # 调拨信息
    from_warehouse_id = db.Column(db.Integer, db.ForeignKey('warehouse.id'), nullable=False)
    to_warehouse_id = db.Column(db.Integer, db.ForeignKey('warehouse.id'), nullable=False)
    spare_part_id = db.Column(db.Integer, db.ForeignKey('spare_part.id'), nullable=False)
    from_location_id = db.Column(db.Integer, db.ForeignKey('warehouse_location.id'))
    to_location_id = db.Column(db.Integer, db.ForeignKey('warehouse_location.id'))
    quantity = db.Column(db.Integer, nullable=False)
    
    # 审批状态
    status = db.Column(db.String(20), default='pending')  # pending/approved/in_transit/completed/cancelled
    approval_required = db.Column(db.Boolean, default=True)
    reason = db.Column(db.Text)
    
    # 关联出入库单
    outbound_order_id = db.Column(db.Integer, db.ForeignKey('outbound_order.id'))
    inbound_order_id = db.Column(db.Integer, db.ForeignKey('inbound_order.id'))
    
    # 审计字段
    created_by = db.Column(db.Integer, db.ForeignKey('user.id'))
    approved_by = db.Column(db.Integer, db.ForeignKey('user.id'))
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    approved_at = db.Column(db.DateTime)
    completed_at = db.Column(db.DateTime)
```

#### 2.3.2 调拨业务流程

```
1. 发起调拨申请 → 创建 TransferOrder (status=pending)
2. 审批通过 → status=approved
3. 执行出库 → 创建 OutboundOrder (outbound_type=transfer)
              → status=in_transit
4. 目标仓库确认收货 → 创建 InboundOrder (inbound_type=transfer)
                      → status=completed
                      → 触发 InventoryRecord 联动更新
```

### F007：移动端扫码操作（PWA）

#### 2.7.1 技术方案

```
前端: React + PWA (Service Worker)
扫码: jsQR 或 BarcodeDetector API
入口: /m/ 路由（单独的 React 页面）
```

#### 2.7.2 扫码入库流程

```
打开扫码页面 → 摄像头扫描条形码
    │
    ▼
调用 /api/spare-parts/search-by-barcode
    │
    ▼
展示备件信息（名称、当前库存、图片）
    │
    ▼
用户填写：入库数量、仓库、货位
    │
    ▼
提交创建 InboundOrder
    │
    ▼
显示成功，继续扫下一个
```

### F009：仪表盘重构

#### 2.9.1 数据指标卡

| 指标 | 数据来源 | 更新频率 |
|------|----------|---------|
| 今日入库笔数/数量 | InboundOrder（当日） | 实时 |
| 今日出库笔数/数量 | OutboundOrder（当日） | 实时 |
| 低库存备件数 | SparePart.stock_status='low' | 实时 |
| 缺货备件数 | SparePart.stock_status='out' | 实时 |
| 待审批单据数 | InboundOrder+OutboundOrder+TransferOrder status=pending | 实时 |
| 本月采购金额 | PurchaseOrder 当月 | 日更新 |
| 库存总价值 | SUM(InventoryRecord.total_amount) | 日更新 |

#### 2.9.2 图表设计

1. **库存状态分布**（环形图）：正常/低库存/缺货/超储数量占比
2. **近30天收发趋势**（折线图）：入库数量 vs 出库数量
3. **TOP 10 消耗备件**（水平柱状图）：本月消耗排行
4. **仓库利用率**（进度条列表）：各仓库容量使用率
5. **预警列表**（表格）：最新 10 条告警
