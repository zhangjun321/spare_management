# 数据库字段全面修复报告

## 修复时间
2026-03-18

## 问题背景
用户反馈模型字段与数据库表不一致，导致频繁出现错误。要求全面检查并修复所有字段。

## 发现的问题

### 1. Category 模型字段不一致
**问题**: 
- 模型只定义了 `is_active` (Boolean) 字段
- 表单查询使用 `status='active'` 
- 导致错误：`Entity namespace for "Category" has no property "status"`

**修复方案**:
- ✓ 在 Category 表中添加 `status` 字段 (VARCHAR(20))
- ✓ 在 Category 模型中添加 `status` 字段定义
- ✓ 同步数据：根据 `is_active` 设置 `status` 值
- ✓ 修复表单查询：统一使用 `is_active=True`

### 2. Supplier 模型字段不一致
**问题**:
- 模型只定义了 `status` (VARCHAR) 字段
- 表单查询使用 `is_active=True`
- 导致错误：`Entity namespace for "supplier" has no property "is_active"`

**修复方案**:
- ✓ 在 Supplier 表中添加 `is_active` 字段 (BOOLEAN)
- ✓ 在 Supplier 模型中添加 `is_active` 字段定义
- ✓ 同步数据：根据 `status` 设置 `is_active` 值
- ✓ 修复表单查询：统一使用 `is_active=True`

### 3. SparePartSearchForm 查询字段错误
**问题**:
- 使用 `Category.query.filter_by(status='active')` 但 Category 模型没有 status 字段
- 使用 `Supplier.query.filter_by(status='active')` 但 Supplier 模型没有 status 字段

**修复方案**:
- ✓ 修改为 `Category.query.filter_by(is_active=True)`
- ✓ 修改为 `Supplier.query.filter_by(is_active=True)`

## 执行的修复操作

### 数据库修复
1. **Category 表**
   - 添加字段：`status VARCHAR(20) DEFAULT 'active' COMMENT '状态 (active/inactive)'`
   - 同步数据：`UPDATE category SET status = CASE WHEN is_active = TRUE THEN 'active' ELSE 'inactive' END WHERE status IS NULL`

2. **Supplier 表**
   - 添加字段：`is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用'`
   - 同步数据：`UPDATE supplier SET is_active = CASE WHEN status = 'active' THEN TRUE ELSE FALSE END WHERE is_active IS NULL`

### 模型文件修复
1. **app/models/category.py**
   - 添加字段定义：`status = db.Column(db.String(20), nullable=False, default='active', comment='状态')`

2. **app/models/supplier.py**
   - 添加字段定义：`is_active = db.Column(db.Boolean, nullable=False, default=True, comment='是否启用')`

### 表单文件修复
1. **app/forms/spare_parts.py**
   - SparePartSearchForm.__init__: 将 Category 和 Supplier 查询统一改为使用 `is_active=True`

## 字段命名规范统一

为了保持一致性，系统采用以下规范：

### 状态字段命名
- **主要字段**: `is_active` (Boolean) - 用于表示记录是否启用/激活
- **辅助字段**: `status` (String) - 用于表示复杂的状态（如 'active', 'inactive', 'pending' 等）

### 各表状态字段配置
| 表名 | is_active | status | 说明 |
|------|-----------|--------|------|
| category | ✓ | ✓ | 双字段兼容 |
| supplier | ✓ | ✓ | 双字段兼容 |
| spare_part | ✓ | ✓ | 使用 is_active |
| equipment | ✓ | ✓ | 使用 is_active |
| warehouse | ✓ | - | 仅使用 is_active |
| department | ✓ | - | 仅使用 is_active |
| user | ✓ | - | 仅使用 is_active |
| batch | - | ✓ | 仅使用 status |
| maintenance_order | - | ✓ | 仅使用 status |
| 其他业务表 | - | ✓ | 使用 status 表示业务状态 |

## 建议

1. **查询规范**: 所有查询启用/禁用状态的，统一使用 `is_active` 字段
2. **业务状态**: 表示业务流转状态的，使用 `status` 字段
3. **新增模型**: 遵循上述字段命名规范
4. **定期验证**: 运行 `python scripts/verify_all_models.py` 验证模型一致性
