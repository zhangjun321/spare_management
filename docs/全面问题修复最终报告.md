### 修复 1: User.managed_warehouses

**问题文件**: `app/models/user.py`

**修复前**:
```python

    # 关系
    department = db.relationship('Department', foreign_keys=[department_id])
    role = db.relationship('Role', foreign_keys=[role_id], backref='role_users')
```

**修复后**:
```python

    # 关系
    department = db.relationship('Department', foreign_keys=[department_id])
    role = db.relationship('Role', foreign_keys=[role_id], backref='role_users')
    managed_warehouses = db.relationship('Warehouse', foreign_keys='Warehouse.manager_id', back_populates='manager')
```

### 修复 5: InventoryRecord 关系配置冲突

**问题文件**: `app/models/inventory_record.py`

**错误**: `ArgumentError: Error creating backref 'inventory_records' on relationship 'InventoryRecord.spare_part': property of that name exists on mapper 'Mapper[SparePart(spare_part)]'`

**原因**:
- `InventoryRecord` 使用了 `backref='inventory_records'`
- 但 `SparePart` 已经定义了 `inventory_records` 关系
- 导致 backref 和 back_populates 冲突

**修复**:
```python
# 修复前（错误）:
spare_part = db.relationship('SparePart', foreign_keys=[spare_part_id], backref='inventory_records')
warehouse = db.relationship('Warehouse', foreign_keys=[warehouse_id], backref='inventory_records')
warehouse_location = db.relationship('WarehouseLocation', foreign_keys=[location_id], backref='inventory_records')

# 修复后（正确）:
spare_part = db.relationship('SparePart', foreign_keys=[spare_part_id], back_populates='inventory_records')
warehouse = db.relationship('Warehouse', foreign_keys=[warehouse_id], back_populates='inventory_records')
warehouse_location = db.relationship('WarehouseLocation', foreign_keys=[location_id], back_populates='inventory_records')
```

**验证**:
- ✅ 应用启动正常
- ✅ 无 backref 冲突错误
- ✅ 库存记录关系正常
