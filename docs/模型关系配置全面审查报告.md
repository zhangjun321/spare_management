## 关系配置最佳实践总结

### 1. 双向关系配置规则

**正确示例**:
```python
# 父模型
class Parent(db.Model):
    children = db.relationship('Child', back_populates='parent')

# 子模型
class Child(db.Model):
    parent = db.relationship('Parent', back_populates='children')
```

**错误示例**:
```python
# 父模型
class Parent(db.Model):
    children = db.relationship('Child', back_populates='child')  # 错误：名称不匹配

# 子模型
class Child(db.Model):
    parent = db.relationship('Parent', back_populates='children')  # 错误：名称不匹配
```

### 2. 单向关系配置规则

当只需要在一个方向访问关系时，使用 `foreign_keys`：

```python
class Child(db.Model):
    parent = db.relationship('Parent', foreign_keys=[parent_id])
```

### 3. 外键引用规范

**正确格式**:
```python
db.ForeignKey('table_name.column_name')
```

**常见错误**:
- ❌ `db.ForeignKey('ModelName.id')` - 应该使用表名，不是模型名
- ❌ `db.ForeignKey('warehouse_shelf.id')` - 表名不存在
- ✅ `db.ForeignKey('warehouse_rack.id')` - 正确的表名

### 4. backref vs back_populates

**backref** (单向快捷方式):
```python
class Parent(db.Model):
    children = db.relationship('Child', backref='parent')
```

**back_populates** (双向显式):
```python
# 需要两端都定义
class Parent(db.Model):
    children = db.relationship('Child', back_populates='parent')

class Child(db.Model):
    parent = db.relationship('Parent', back_populates='children')
```
