## 一、需求分析

### 1.1 业务背景

当前系统的仓库管理模块存在核心功能缺失问题：
- React 前端有页面但后端 API 未实现
- Flask 模板版本和 React 版本并存，造成混乱
- 缺少库存联动展示功能
- 性能问题严重（N+1 查询）

### 1.2 用户需求

#### 用户故事

**US-001**: 仓库管理员查看仓库列表
- **作为** 仓库管理员
- **我希望** 查看所有仓库的列表，支持分页和筛选
- **以便** 快速找到目标仓库

**US-002**: 仓库管理员创建新仓库
- **作为** 仓库管理员
- **我希望** 通过表单创建新仓库
- **以便** 管理新的仓储空间

**US-003**: 仓库管理员查看仓库详情
- **作为** 仓库管理员
- **我希望** 查看仓库的详细信息和统计数据
- **以便** 了解仓库运营状况

**US-004**: 仓库管理员查看库存
- **作为** 仓库管理员
- **我希望** 查看某个仓库的所有备件库存
- **以便** 掌握库存分布

**US-005**: 仓库管理员删除仓库
- **作为** 仓库管理员
- **我希望** 删除空仓库
- **以便** 保持数据整洁

### 1.3 功能需求

#### FR-001: 仓库列表查询
- 支持分页（默认 20 条/页）
- 支持关键词搜索（名称、编码、地址）
- 支持类型筛选
- 支持启用状态筛选
- 支持多字段排序
- 预加载关联数据（避免 N+1 查询）

#### FR-002: 仓库详情查询
- 返回仓库基本信息
- 返回统计数据（库位数量、库存数量、订单数量等）
- 返回库位列表
- 返回区域列表

#### FR-003: 创建仓库
- 验证必填字段（名称、编码）
- 检查编码唯一性
- 支持可选字段
- 记录创建人
- 自动设置创建时间

#### FR-004: 更新仓库
- 支持全量更新（PUT）
- 支持部分更新（PATCH）
- 检查编码唯一性（排除自身）
- 记录更新人和更新时间

#### FR-005: 删除仓库
- 删除前校验（库位、库存、订单）
- 提供友好的错误提示
- 支持批量删除

#### FR-006: 库存联动展示
- 查看某仓库的所有备件库存
- 支持分页
- 支持搜索和筛选
- 显示备件信息和库位信息

### 1.4 非功能需求

#### 性能要求
- 列表查询响应时间 < 500ms
- 详情查询响应时间 < 200ms
- 支持并发用户数 > 100

#### 安全要求
- 所有 API 需要登录验证
- 操作类 API 需要权限验证
- CSRF 保护
- 输入验证和 SQL 注入防护

#### 可用性要求
- API 可用性 > 99%
- 错误提示清晰友好
- 支持事务回滚

## 二、功能设计

### 2.1 系统架构

```
┌─────────────────────────────────────────────┐
│           React 前端 (Ant Design)            │
├─────────────────────────────────────────────┤
│  WarehouseList  │  WarehouseForm  │  Detail │
└─────────────────────────────────────────────┘
                      ↓ HTTP/JSON
┌─────────────────────────────────────────────┐
│         Flask REST API (Blueprint)           │
├─────────────────────────────────────────────┤
│  api_warehouses.py (10+ API endpoints)       │
└─────────────────────────────────────────────┘
                      ↓ SQLAlchemy
┌─────────────────────────────────────────────┐
│            数据库 (MySQL/PostgreSQL)          │
├─────────────────────────────────────────────┤
│  warehouse | inventory_record | spare_part  │
└─────────────────────────────────────────────┘
```

### 2.2 模块划分

#### 2.2.1 路由层 (Routes)
- **文件**: `app/routes/api_warehouses.py`
- **职责**: 处理 HTTP 请求，参数验证，调用服务层

#### 2.2.2 服务层 (Services)
- **文件**: `app/services/warehouse_service.py`
- **职责**: 业务逻辑，数据验证，事务管理

#### 2.2.3 模型层 (Models)
- **文件**: `app/models/warehouse.py`
- **职责**: 数据映射，关联关系，数据验证

#### 2.2.4 前端服务 (Frontend Services)
- **文件**: `frontend/src/services/warehouse.js`
- **职责**: API 调用，数据转换，错误处理

### 2.3 流程设计

#### 2.3.1 创建仓库流程

```
用户 → 填写表单 → 前端验证 → POST /api/warehouses
                                    ↓
                          后端验证（必填字段、唯一性）
                                    ↓
                          创建仓库记录 → 设置创建人
                                    ↓
                          提交事务 → 返回成功
                                    ↓
                          前端提示 → 跳转列表
```

#### 2.3.2 删除仓库流程

```
用户 → 确认删除 → DELETE /api/warehouses/:id
                                    ↓
                          检查是否有关联数据
                            ├─ 有库位 → 返回错误
                            ├─ 有库存 → 返回错误
                            ├─ 有订单 → 返回错误
                            └─ 无关联 → 执行删除
                                    ↓
                          提交事务 → 返回成功
```

## 三、数据模型

### 3.1 核心模型

#### Warehouse (仓库表)

```python
class Warehouse(db.Model):
    id              # 主键
    name            # 仓库名称
    code            # 仓库编码（唯一）
    type            # 仓库类型
    address         # 仓库地址
    capacity        # 仓库容量
    area            # 仓库面积
    manager_id      # 仓库管理员（外键）
    phone           # 联系电话
    email           # 邮箱
    is_active       # 是否启用
    remark          # 备注
    created_at      # 创建时间
    updated_at      # 更新时间
    created_by      # 创建人（外键）
    updated_by      # 更新人（外键）
    
    # 关联关系
    manager         # 仓库管理员（User）
    locations       # 库位列表（WarehouseLocation）
    zones           # 区域列表（WarehouseZone）
    inventory_records  # 库存记录（InventoryRecord）
    inbound_orders     # 入库单（InboundOrder）
    outbound_orders    # 出库单（OutboundOrder）
```

### 3.2 关联模型

#### InventoryRecord (库存记录表)

```python
class InventoryRecord(db.Model):

    quantity        # 库存数量
    locked_quantity # 锁定数量
    available_quantity  # 可用数量
    stock_status    # 库存状态
```

## 四、API 接口规范

### 4.1 通用规范

#### 响应格式

**成功响应**:
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

**错误响应**:
```json
{
  "success": false,
  "error": "错误描述",
  "code": "错误码"
}
```

**分页响应**:
```json
{
  "success": true,
  "data": [ ... ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 100,
    "pages": 5,
    "has_next": true,
    "has_prev": false
  }
}
```

#### HTTP 状态码

| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | OK | GET/PUT/PATCH 成功 |
| 201 | Created | POST 创建成功 |
| 204 | No Content | DELETE 成功 |
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未登录 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 500 | Internal Server Error | 服务器错误 |
