# 工业设备维护管理系统 - 系统集成指南

## 快速开始

### 1. 数据库初始化

运行初始化脚本：

```bash
python scripts/init_industrial_equipment.py
```

该脚本将：
- 创建所需的数据库表
- 初始化设备类型
- 创建设备示例数据
- 创建示例组件
- 初始化知识库
- 初始化备件数据
- 创建库存记录

### 2. 启动应用

```bash
python run.py
```

应用将在 `http://localhost:5000` 启动

### 3. API端点

工业设备管理系统的API都挂载在 `/api/industrial` 路径下。

#### 设备管理
- `GET /api/industrial/devices` - 获取设备列表
- `POST /api/industrial/devices` - 创建设备
- `GET /api/industrial/devices/<id>` - 获取设备详情
- `GET /api/industrial/device-types` - 获取设备类型
- `GET /api/industrial/devices/<id>/components` - 获取设备组件
- `POST /api/industrial/devices/<id>/components` - 创建设备组件

#### 告警管理
- `GET /api/industrial/alerts` - 获取告警列表
- `GET /api/industrial/alerts/<id>` - 获取告警详情
- `PUT /api/industrial/alerts/<id>/acknowledge` - 确认告警
- `PUT /api/industrial/alerts/<id>/resolve` - 处理告警
- `GET /api/industrial/alerts/statistics` - 获取告警统计

#### 维修工单
- `GET /api/industrial/work-orders` - 获取工单列表
- `POST /api/industrial/work-orders` - 创建工单
- `GET /api/industrial/work-orders/<id>` - 获取工单详情
- `PUT /api/industrial/work-orders/<id>/assign` - 派单
- `PUT /api/industrial/work-orders/<id>/start` - 开始处理
- `PUT /api/industrial/work-orders/<id>/complete` - 完成工单
- `GET /api/industrial/work-orders/<id>/logs` - 获取工单日志
- `POST /api/industrial/work-orders/<id>/logs` - 创建工单日志

#### 知识库
- `GET /api/industrial/knowledge` - 获取知识库列表
- `POST /api/industrial/knowledge` - 创建知识库条目
- `GET /api/industrial/knowledge/<id>` - 获取知识库详情

#### 备件管理
- `GET /api/industrial/spare-parts` - 获取备件列表
- `POST /api/industrial/spare-parts` - 创建备件
- `GET /api/industrial/spare-parts/inventory` - 获取库存

#### 统计分析
- `GET /api/industrial/dashboard/overview` - 获取仪表板概览

### 4. 示例数据

## 告警机制

### 告警级别

| 级别 | 颜色 | 说明 | 响应时间要求 |
|------|------|------|--------------|
| 紧急 | 红色 | 严重故障 | 5分钟内响应 |
| 重要 | 橙色 | 重要故障 | 30分钟内响应 |
| 一般 | 黄色 | 警告信息 | 2小时内响应 |
| 提示 | 蓝色 | 一般提示 | 24小时内关注 |

### 告警流程

1. **告警触发**: 设备数据超过阈值或状态异常
2. **告警去重**: 判断是否为相同告警源和类型
3. **告警通知**: 根据级别选择通知方式
4. **告警确认**: 维护人员查看并确认
5. **工单创建**: 从告警快速生成工单
6. **维修处理**: 工程师现场处理
7. **故障解决**: 确认修复，关闭告警

### PLC接入示例

#### 西门子 S7 系列

```python
# 使用 snap7 库连接
import snap7

client = snap7.client.Client()
client.connect('192.168.1.100', 0, 1)

# 读取数据
data = client.db_read(1, 0, 10)

# 写入数据
client.db_write(1, 0, b'\x01\x02\x03')

client.disconnect()
```

#### Modbus TCP

```python
from pymodbus.client import ModbusTcpClient

client = ModbusTcpClient('192.168.1.100')
client.connect()

# 读取保持寄存器
result = client.read_holding_registers(0, 10, unit=1)

# 写入单个寄存器
client.write_register(0, 1234, unit=1)

client.close()
```

## 系统优化策略

### 降低设备故障率

1. **预测性维护**: 基于历史数据分析，提前发现潜在故障
2. **状态监控**: 实时监控设备运行状态，及时发现异常
3. **定期保养**: 建立预防性维护计划，定期保养设备
4. **备件管理**: 优化备件库存，确保维修所需备件

### 减少维修时间

1. **快速响应**: 实时告警通知，快速响应
2. **知识库**: 建立故障知识库，提供解决方案
3. **备件准备**: 提前准备备件，减少等待时间
4. **标准化流程**: 建立标准维修流程，提高效率

## 常见问题

### 数据库连接问题

检查数据库配置：
- 确认MySQL服务正常运行
- 检查用户名和密码
- 验证数据库名是否正确

### API请求失败

检查以下项目：
- 确认应用正常运行
- 检查用户登录状态
- 验证请求参数格式

### 实时数据更新延迟

检查：
- WebSocket连接状态
- 服务器负载情况
- 网络连接质量
