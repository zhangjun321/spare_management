# 库存状态定义更新

## 📋 更新说明

**时间**：2026-03-19

**更新内容**：优化库存状态的定义，将"缺货"从"低库存"中独立出来，形成更精确的四级状态体系。

---

## 🎯 更新前后的区别

### 更新前（三级状态）

```
1. normal（正常）：min_stock < current_stock < max_stock
2. low（低库存）：current_stock <= min_stock  ❌ 包含 0
3. overstocked（超储）：current_stock >= max_stock
```

**问题**：
- 库存为 0 时也被判断为"低库存"
- 无法区分"缺货"和"低库存"两种不同程度的预警
- 不利于优先级管理

### 更新后（四级状态）

```
1. out（缺货）：current_stock = 0              🔴 紧急
2. low（低库存）：0 < current_stock <= min_stock  🟠 预警
3. normal（正常）：min_stock < current_stock < max_stock ✅ 正常
4. overstocked（超储）：current_stock >= max_stock  🟡 预警
```

**优势**：
- ✅ 明确区分"缺货"和"低库存"
- ✅ 更精确的库存预警级别
- ✅ 便于优先级管理（缺货优先处理）
- ✅ 符合业务实际需求

---

## 📊 库存状态详细定义

### 1. 🔴 缺货（out）

**定义**：当前库存 = 0

**条件**：
```python
current_stock == 0
```

**说明**：
- 库存已完全耗尽
- 最紧急的状态
- 需要立即补货
- 可能已影响业务

**图标**：❌ 红色叉号  
**颜色**：红色（bg-danger）  
**优先级**：P0（最高）

**示例**：
```
最低库存：10 个
当前库存：0 个
状态：🔴 缺货
```

---

### 2. 🟠 低库存（low）

**定义**：0 < 当前库存 ≤ 最低库存

**条件**：
```python
current_stock > 0 AND current_stock <= min_stock
```

**说明**：
- 库存已低于安全线
- 需要尽快安排补货
- 但还能维持短期运营

**图标**：⚠️ 红色感叹号  
**颜色**：红色（bg-danger）  
**优先级**：P1（高）

**示例**：
```
最低库存：10 个
当前库存：5 个
状态：🟠 低库存
```

---

### 3. ✅ 正常（normal）

**定义**：最低库存 < 当前库存 < 最高库存

**条件**：
```python
current_stock > min_stock AND (max_stock is None OR current_stock < max_stock)
```

**说明**：
- 库存在合理范围内
- 不需要补货
- 不需要处理

**图标**：✓ 绿色对勾  
**颜色**：绿色（bg-success）  
**优先级**：P3（低）

**示例**：
```
最低库存：10 个
最高库存：100 个
当前库存：50 个
状态：✅ 正常
```

---

### 4. 🟡 超储（overstocked）

**定义**：当前库存 ≥ 最高库存

**条件**：
```python
max_stock is not None AND current_stock >= max_stock
```

**说明**：
- 库存已超过警戒线
- 库存积压，占用资金
- 可能需要促销或调拨

**图标**：⚠️ 黄色感叹号  
**颜色**：黄色（bg-warning）  
**优先级**：P2（中）

**示例**：
```
最低库存：10 个
最高库存：100 个
当前库存：120 个
状态：🟡 超储
```

## 🔧 代码实现

### 模型方法

**文件**：`app/models/spare_part.py`

```python
def update_stock_status(self):
    """更新库存状态"""
    if self.current_stock == 0:
        self.stock_status = 'out'  # 缺货
    elif self.current_stock <= self.min_stock:
        self.stock_status = 'low'  # 低库存
    elif self.max_stock and self.current_stock >= self.max_stock:
        self.stock_status = 'overstocked'  # 超储
    else:
        self.stock_status = 'normal'  # 正常
```

### SQL 更新语句

```sql
UPDATE spare_part 
SET stock_status = CASE
    WHEN current_stock = 0 THEN 'out'
    WHEN current_stock <= min_stock THEN 'low'
    WHEN max_stock IS NOT NULL AND current_stock >= max_stock THEN 'overstocked'
    ELSE 'normal'
END;
```
