# 统一间距系统文档

## 📋 设计理念

建立统一的间距系统，确保所有页面组件之间的间距一致、舒适、美观，避免间距过大或过小的问题。

## 🎯 间距层级系统

### 定义 5 个层级的间距变量

```css
:root {
    --spacing-xs: 0.5rem;   /* 8px  - 微小间距 */
    --spacing-sm: 1rem;     /* 16px - 小间距 */
    --spacing-md: 1.5rem;   /* 24px - 中等间距 */
    --spacing-lg: 2rem;     /* 32px - 大间距 */
    --spacing-xl: 2.5rem;   /* 40px - 超大间距 */
}
```

### 间距使用场景

#### 1. **微小间距 (--spacing-xs: 8px)**
- 图标与文字之间
- 紧密相关的元素
- Footer 列表项间距
- 徽章内边距

#### 2. **小间距 (--spacing-sm: 16px)**
- 表格底部间距
- 顶部导航栏内边距
- Footer 标题间距
- 按钮内边距
- 列表项间距

#### 3. **中等间距 (--spacing-md: 24px)** ⭐ 最常用
- 主内容区域内边距
- 卡片底部间距
- 顶部导航栏底部间距
- 表格内边距
- Footer 底部区域上边距
- 行（row）间距

#### 4. **大间距 (--spacing-lg: 32px)**
- Footer 上下内边距
- 区块之间的间距
- 页面标题间距

#### 5. **超大间距 (--spacing-xl: 40px)**
- 超大区块间距
- 特殊场景区距

## 📦 组件间距应用

### 1. 主内容区域
```css
.main-content {
    padding: var(--spacing-md);        /* 24px */
    min-height: calc(100vh - 180px);
}
```

### 2. 顶部导航栏
```css
.top-navbar {
    padding: var(--spacing-sm) var(--spacing-md);  /* 16px 32px */
    margin-bottom: var(--spacing-md);              /* 24px */
}
```

### 3. 卡片组件
```css
.stat-card {
    margin-bottom: var(--spacing-md);  /* 24px */
}

.stat-card .card-body {
    padding: calc(var(--spacing-md) + 0.25rem);  /* 28px */
}
```

### 4. 表格容器
```css
.table-responsive {
    padding: var(--spacing-md);        /* 24px */
    margin-bottom: var(--spacing-sm);  /* 16px */
}
```

### 5. Footer
```css
.footer {
    padding: var(--spacing-lg) 0 var(--spacing-sm);  /* 32px 0 16px */
}

.footer h5 {
    margin-bottom: var(--spacing-sm);                /* 16px */
    padding-bottom: calc(var(--spacing-xs) + 0.25rem); /* 12px */
}

.footer-links li {
    margin-bottom: calc(var(--spacing-xs) + 0.25rem);  /* 12px */
}

.footer-bottom {
    padding-top: var(--spacing-sm);    /* 16px */
    margin-top: var(--spacing-md);     /* 24px */
}
```

## 🎯 间距原则

### 1. **一致性原则**
- 相同层级的元素使用相同的间距
- 避免随意使用固定值
- 优先使用间距变量

### 2. **层次性原则**
- 相关元素：使用小间距（--spacing-xs, --spacing-sm）
- 独立组件：使用中间距（--spacing-md）
- 区块分隔：使用大间距（--spacing-lg, --spacing-xl）

### 3. **呼吸感原则**
- 不要过于紧密（至少 8px）
- 不要过于稀疏（最多 40px）
- 保持视觉舒适

### 4. **响应式原则**
- 移动端适当减小间距
- 桌面端保持标准间距
- 使用媒体查询调整

## 📱 响应式间距

### 移动端优化（≤768px）
```css
@media (max-width: 768px) {
    .main-content {
        padding: var(--spacing-sm);  /* 16px */
    }
    
    .footer {
        padding: var(--spacing-md) 0 calc(var(--spacing-xs) + 0.25rem);
    }
}
```

### 间距调整策略
- 主内容区：24px → 16px（减少 33%）
- Footer 顶部：32px → 24px（减少 25%）
- 保持内部间距不变

## 🔧 使用指南

### 1. 开发新页面时
```html
<!-- 使用 Bootstrap 间距 + 自定义间距 -->
<div class="mb-3 mb-spacing-md">内容</div>
```

### 2. 调整现有页面
```html
<!-- 替换固定值为变量 -->
<!-- 前 -->
<div style="margin-bottom: 24px">

<!-- 后 -->
<div class="mb-spacing-md">
```

### 3. 自定义场景区距
```css
.custom-section {
    margin-bottom: var(--spacing-lg);  /* 32px */
}
```

## 💡 最佳实践

### ✅ 推荐做法
1. 使用间距变量
2. 遵循间距层级
3. 保持视觉一致性
4. 考虑响应式适配

### ❌ 避免做法
1. 使用固定像素值
2. 随意创建新间距
3. 间距过大或过小
4. 忽略移动端适配
