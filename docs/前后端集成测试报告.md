## 问题分析

### 核心问题：CSRF 保护

**现象**:
1. 登录接口返回 400 错误
2. 所有需要认证的 API 返回 HTML 而非 JSON
3. 前端请求拦截器配置的 CSRF token 获取方式在测试脚本中未实现

**根本原因**:
```python
# 前端代码正确配置了 CSRF token
apiClient.interceptors.request.use((config) => {
  const csrfToken = document.cookie
    .split('; ')
    .find((row) => row.startsWith('csrf_token='))
    ?.split('=')[1]
  
  if (csrfToken) {
    config.headers['X-CSRFToken'] = csrfToken
  }
  return config
})
```

但测试脚本中未实现相同的逻辑。

### 解决方案

#### 方案 1: 在测试脚本中实现 CSRF token 处理

```python
def login(self):
    # 获取登录页面和 CSRF token
    login_page = self.session.get(f"{BASE_URL}/login")
    
    # 从 cookie 中提取 CSRF token
    csrf_token = self.session.cookies.get('csrf_token')
    
    # 在后续请求中添加 CSRF token
    headers = {'X-CSRFToken': csrf_token}
    response = self.session.post(f"{BASE_URL}/login", 
                                  data={'username': 'admin', 'password': 'admin123'},
                                  headers=headers)
```

#### 方案 2: 测试环境临时豁免 CSRF

```python
# 在 app/__init__.py 或测试配置中
if app.config['DEBUG']:
    from app.extensions import csrf
    csrf.exempt(auth_bp)  # 豁免认证模块
    csrf.exempt(api_bp)   # 豁免 API 模块
```

#### 方案 3: 使用浏览器自动化测试

使用 Selenium 或 Playwright 进行真实的浏览器测试，自动处理 CSRF 和认证。
