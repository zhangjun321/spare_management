## 🔧 更新逻辑

### 1. 缺货（out）
```python
current_stock = 0
```

### 2. 低库存（low）
```python
# 生成 1 到 min_stock 之间的随机数
current_stock = random.randint(1, min_stock)
```

### 3. 正常（normal）
```python
# 生成 min_stock+1 到 max_stock-1 之间的随机数
current_stock = random.randint(min_stock + 1, max_stock - 1)
```

### 4. 超储（overstocked）
```python
# 生成 max_stock 到 max_stock*1.5 之间的随机数
current_stock = random.randint(max_stock, int(max_stock * 1.5))
```

## 🎯 业务意义

### 合理的库存结构

1. **90% 正常库存**
   - 大部分备件库存合理
   - 资金利用效率高
   - 运营稳定

2. **2% 缺货 + 3% 低库存**
   - 少量预警是正常的
   - 提醒需要补货
   - 便于优先级管理

3. **5% 超储**
   - 模拟真实场景
   - 可能需要处理积压
   - 优化库存结构

## ⚠️ 注意事项

### 1. 数据随机性

- 每次运行脚本会生成不同的随机数据
- 如需固定数据，请设置随机种子
- 建议在生产环境使用前备份数据

### 2. 库存阈值

- 如果 min_stock 或 max_stock 为 0，会自动设置为默认值
- min_stock 默认：10
- max_stock 默认：100

### 3. 状态更新

- 更新库存后会自动调用 `update_stock_status()`
- 确保库存状态与库存数量一致
