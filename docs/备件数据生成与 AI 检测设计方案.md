# 备件管理数据生成与 AI 检测功能设计方案

## 📋 项目概述

### 1.1 项目背景
为完善备件管理系统的数据基础，提升数据质量，本项目将：
- 使用硅基流动 AI API 生成 100 条真实、准确的备件数据
- 建立标准化的三级分类体系
- 创建规范的备件编码规则
- 实现 AI 智能检测与修复建议功能

### 1.2 技术栈
- **AI 平台**：硅基流动 (SiliconFlow)
- **AI 模型**：Qwen/Qwen2.5-72B-Instruct（文本生成）
- **后端框架**：Flask + SQLAlchemy
- **数据库**：SQLite/MySQL
- **前端框架**：Bootstrap 5

## 📊 第一部分：三级分类标准体系

### 2.1 分类原则
采用"大类 - 中类 - 小类"三级分类结构，遵循以下原则：
- **系统性**：覆盖工业设备全品类备件
- **专业性**：符合行业标准和实际应用场景
- **扩展性**：预留扩展空间，便于后续补充
- **唯一性**：每个小类有明确的编码和定义

### 2.2 分类代码结构
```
分类代码：XX.XX.XX
├── 第一位大类：1 位字母（A-Z）
├── 第二位中类：2 位数字（01-99）
└── 第三位小类：2 位数字（01-99）
```

## 🔢 第二部分：标准化备件编码规则

### 3.1 编码结构设计
```
备件编码：XXXX-XXXXXX-XXX-XXX
          │    │       │    └─ 流水号 (3 位)
          │    │       └────── 规格代码 (3 位)
          │    └────────────── 分类代码 (6 位)
          └─────────────────── 品牌代码 (4 位)
```

### 3.2 编码规则详解

#### (1) 品牌代码 (4 位)
```
国际品牌：
  SKF0 - SKF 轴承
  FAG0 - FAG 轴承
  NSK0 - NSK 轴承
  TIMK - Timken 轴承
  INAF - INA 轴承
  SIME - Siemens 电气
  ABBC - ABB 电气
  SCHN - Schneider 电气
  OMRN - Omron 电气
  MITS - Mitsubishi 电气

国内品牌：
  HRB0 - 哈尔滨轴承
  ZWZ0 - 瓦房店轴承
  LYC0 - 洛阳轴承
  C&U0 - 人本轴承
  TCL0 -  TCL 电气
  CHNT - 正泰电气
  DELIX - 德力西电气
  HIKR0 - 海康电气
  INOV - 汇川技术
  ESTU - 汇川伺服

通用品牌：
  GEN0 - 通用件
  CUST - 定制件
  OEM0 - OEM 件
  UNKN - 未知品牌
```

#### (2) 分类代码 (6 位)
采用三级分类代码，格式：`XX.XX.XX`
- 示例：`A.01.01` = 机械传动件 - 轴承类 - 深沟球轴承

#### (3) 规格代码 (3 位)
按尺寸系列编码：
```
001-099: 小型规格 (0-50mm)
100-199: 中型规格 (50-200mm)
200-299: 大型规格 (200-500mm)
300-399: 特大型规格 (500mm 以上)
```

#### (4) 流水号 (3 位)
```
001-999: 同类备件顺序号
```

## 🤖 第三部分：AI 检测功能设计

### 4.1 检测功能架构

```
┌─────────────────────────────────────────┐
│         AI 检测功能模块                   │
├─────────────────────────────────────────┤
│  1. 数据完整性检测                      │
│  2. 命名规范性检测                      │
│  3. 分类准确性检测                      │
│  4. 编码正确性检测                      │
│  5. 参数合理性检测                      │
│  6. 供应商可信度检测                    │
│  7. 价格异常检测                        │
│  8. 重复数据检测                        │
└─────────────────────────────────────────┘
```

### 4.2 检测规则详细设计

#### 规则 1：数据完整性检测
```python
检测项：
- 备件名称：不能为空，长度 5-100 字符
- 规格型号：不能为空，符合命名规范
- 分类代码：必须为有效的三级分类
- 单位：必须为标准单位（个、套、米、千克等）
- 单价：必须为正数
- 库存数量：必须为非负数
- 供应商：不能为空
- 品牌：不能为空

AI 修复建议：
- 自动补全缺失的标准单位
- 自动格式化不规范的名称
- 提示补充缺失的必要字段
```

#### 规则 2：命名规范性检测
```python
检测项：
- 名称格式：品牌 + 品类 + 关键参数
- 禁止使用特殊字符（除-_.外）
- 禁止使用模糊词汇（如"优质"、"高级"）
- 必须包含关键规格参数

示例：
❌ 错误：优质轴承
✅ 正确：SKF 深沟球轴承 6205-2RS

AI 修复建议：
- 自动提取规格参数补充到名称
- 移除不规范的特殊字符
- 替换模糊词汇为具体参数
```

#### 规则 3：分类准确性检测
```python
检测项：
- 备件名称与分类是否匹配
- 规格型号与分类是否匹配
- 使用场景与分类是否匹配

AI 检测逻辑：
1. 提取备件名称关键词
2. 匹配分类关键词库
3. 计算分类置信度
4. 置信度 < 80% 时给出建议

示例：
备件：SKF 6205 轴承
当前分类：B.02.01 (液压阀类) ❌
建议分类：A.01.01 (深沟球轴承) ✅
置信度：95%
```

#### 规则 4：编码正确性检测
```python
检测项：
- 编码格式是否符合规范
- 品牌代码是否有效
- 分类代码是否匹配
- 流水号是否重复

AI 修复建议：
- 自动生成正确的编码格式
- 纠正错误的品牌代码
- 重新分配重复的流水号
```

#### 规则 5：参数合理性检测
```python
检测项：
- 尺寸参数是否在合理范围
- 重量与尺寸是否匹配
- 价格与品牌是否匹配
- 库存与安全库存是否合理

AI 检测逻辑：
1. 建立参数知识库
2. 对比同类备件参数
3. 识别异常值（超出 3 倍标准差）
4. 给出合理范围建议

示例：
备件：普通螺栓 M10
单价：500 元/个 ❌ (异常高价)
市场均价：2-5 元/个
建议：核实价格或更换供应商
```

#### 规则 6：供应商可信度检测
```python
检测项：
- 供应商名称是否规范
- 供应商资质是否有效
- 供应商品牌授权情况
- 历史供货质量评分

AI 检测逻辑：
1. 查询供应商数据库
2. 验证供应商资质
3. 评估供货历史
4. 计算可信度评分

供应商分级：
- A 级：授权代理商，质量稳定
- B 级：正规供应商，偶尔问题
- C 级：新供应商，需考察
- D 级：有不良记录，慎用
```

#### 规则 7：价格异常检测
```python
检测项：
- 与历史采购价对比
- 与市场均价对比
- 与同类备件对比
- 价格波动趋势

AI 检测逻辑：
1. 收集历史价格数据
2. 查询市场参考价格
3. 计算价格偏离度
4. 标记异常价格

价格偏离度计算：
偏离度 = |当前价 - 参考价| / 参考价 × 100%

预警级别：
- 正常：偏离度 < 20%
- 关注：20% ≤ 偏离度 < 50%
- 警告：50% ≤ 偏离度 < 100%
- 异常：偏离度 ≥ 100%
```

#### 规则 8：重复数据检测
```python
检测项：
- 名称重复检测
- 规格型号重复检测
- 编码重复检测
- 相似备件检测

AI 检测逻辑：
1. 计算文本相似度（余弦相似度）
2. 提取关键参数对比
3. 识别重复和相似备件
4. 给出合并建议

相似度阈值：
- 完全重复：相似度 ≥ 95%
- 高度相似：80% ≤ 相似度 < 95%
- 可能重复：60% ≤ 相似度 < 80%
```

### 5.2 数据生成策略

```python
# scripts/generate_spare_parts_data.py
import requests
import json
from datetime import datetime

class SparePartsGenerator:
    def __init__(self, api_key):
        self.api_key = api_key
        self.api_url = "https://api.siliconflow.cn/v1/chat/completions"
        self.model = "Qwen/Qwen2.5-72B-Instruct"
    
    def generate_parts(self, category, count):
        """生成指定分类的备件数据"""
        prompt = self._build_prompt(category, count)
        
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "model": self.model,
            "messages": [
                {"role": "system", "content": "你是工业备件数据专家，负责生成准确、规范的备件数据。"},
                {"role": "user", "content": prompt}
            ],
            "temperature": 0.7,
            "max_tokens": 4000
        }
        
        response = requests.post(self.api_url, headers=headers, json=payload)
        result = response.json()
        
        # 解析 AI 生成的数据
        parts_data = json.loads(result['choices'][0]['message']['content'])
        
        return parts_data
    
    def _build_prompt(self, category, count):
        """构建 AI 提示词"""
        return f"""
        请生成{count}条{category}类备件数据，要求：
        
        1. 数据字段：
           - 备件名称（格式：品牌 + 品类 + 型号）
           - 规格型号（具体参数）
           - 分类代码（按三级分类标准）
           - 单位（个/套/米/千克等）
           - 单价（合理市场价，人民币）
           - 供应商（真实供应商名称）
           - 品牌
           - 材质
           - 重量（kg）
           - 安全库存
        
        2. 质量要求：
           - 品牌分布合理（国际品牌 60%，国内品牌 30%，通用 10%）
           - 价格符合市场实际
           - 供应商为真实存在的正规供应商
           - 规格型号符合行业标准
        
        3. 输出格式：纯 JSON 数组
        
        示例：
        [
          {{
            "part_name": "SKF 深沟球轴承",
            "spec_model": "6205-2RS",
            "category_code": "A.01.01",
            "unit": "个",
            "unit_price": 45.00,
            "supplier": "上海 SKF 授权经销商",
            "brand": "SKF",
            "material": "轴承钢",
            "weight": 0.15,
            "safety_stock": 20

        ]
        """
```

```python
# scripts/ai_parts_checker.py
import requests
import json

class AIPartsChecker:
    def __init__(self, api_key):
        self.api_key = api_key
        self.api_url = "https://api.siliconflow.cn/v1/chat/completions"
        self.model = "Qwen/Qwen2.5-72B-Instruct"
    
    def check_part(self, part_data):
        """检测单个备件"""
        prompt = self._build_check_prompt(part_data)
        
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "model": self.model,
            "messages": [
                {"role": "system", "content": "你是工业备件数据质量专家，负责检测和修复备件数据问题。"},
                {"role": "user", "content": prompt}
            ],
            "temperature": 0.3,
            "max_tokens": 2000
        }
        
        response = requests.post(self.api_url, headers=headers, json=payload)
        result = response.json()
        
        # 解析检测结果
        check_result = json.loads(result['choices'][0]['message']['content'])
        
        return check_result
    
    def _build_check_prompt(self, part_data):
        """构建检测提示词"""
        return f"""
        请检测以下备件数据的准确性和规范性：
        
        备件信息：
        {json.dumps(part_data, ensure_ascii=False, indent=2)}
        
        检测项：
        1. 数据完整性（必填字段是否完整）
        2. 命名规范性（名称格式是否正确）
        3. 分类准确性（分类代码是否匹配）
        4. 编码正确性（编码格式是否规范）
        5. 参数合理性（尺寸、重量、价格是否合理）
        6. 供应商可信度（供应商是否正规）
        
        输出格式：
        {{
          "is_valid": true/false,
          "score": 0-100,
          "issues": [
            {{
              "type": "问题类型",
              "description": "问题描述",
              "severity": "高/中/低",
              "suggestion": "修复建议",
              "auto_fix": "自动修复值（如有）"
            }}
          ],
          "fixed_data": {{修复后的完整数据}}
        }}
        """
```
