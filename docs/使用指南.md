# 备件数据生成与 AI 检测 - 使用指南

## 📋 概述

本工具使用硅基流动 AI API 实现：
1. 自动生成 100 条真实、准确的备件数据
2. 建立标准化三级分类体系
3. 创建规范的备件编码
4. AI 智能检测数据质量并给出修复建议

### 分类体系

```
A 类 - 机械传动件 (25 条)
  - 轴承类、齿轮类、皮带链条类、联轴器类、导轨丝杆类

B 类 - 液压气动件 (20 条)
  - 液压泵类、液压阀类、液压缸类、气动元件、密封件

C 类 - 电气元件 (20 条)
  - 开关类、继电器类、传感器类、变频器类、电线电缆

D 类 - 紧固件 (15 条)
  - 螺栓类、螺母类、螺钉类、垫圈类、挡圈销类

E 类 - 润滑密封件 (10 条)
  - 润滑油品、润滑脂类、润滑装置、机械密封、密封垫片

F 类 - 过滤净化件 (10 条)
  - 过滤器类、滤芯类、滤网类、除尘器、净化装置
```

### 备件编码规则

```
编码格式：XXXX-XXXXXX-XXX-XXX
          │    │       │    └─ 流水号 (3 位)
          │    │       └────── 规格代码 (3 位)
          │    └────────────── 分类代码 (6 位)
          └─────────────────── 品牌代码 (4 位)

示例：SKF0-A.01.01-050-001
- SKF0: SKF 品牌
- A.01.01: 机械传动件 - 轴承类 - 深沟球轴承
- 050: 中型规格
- 001: 第 1 个同类备件
```

## 🔍 AI 检测功能

### 8 项检测规则

1. **数据完整性检测**
   - 检查必填字段是否完整
   - 验证字段长度和格式

2. **命名规范性检测**
   - 名称格式：品牌 + 品类 + 关键参数
   - 禁止特殊字符和模糊词汇

3. **分类准确性检测**
   - 备件名称与分类代码匹配度
   - 给出分类置信度评分

4. **编码正确性检测**
   - 编码格式验证
   - 品牌代码和分类代码校验

5. **参数合理性检测**
   - 重量、尺寸、材质合理性
   - 安全库存是否适当

6. **供应商可信度检测**
   - 供应商资质验证
   - 信用评级（A/B/C/D）

7. **价格合理性检测**
   - 与市场价对比
   - 计算价格偏离度

8. **重复数据检测**
   - 相似度分析
   - 识别重复和相似备件

### 检测报告

检测报告包含：
- 总体统计（通过率、平均分）
- 各项检测详细结果
- 问题清单和修复建议
- 自动修复后的数据

## ⚙️ 高级用法

### 自定义生成数量

编辑 `generate_spare_parts_data.py`，修改 categories 列表：

```python
categories = [
    ("A", "机械传动件", 50),  # 生成 50 条
    ("B", "液压气动件", 30),  # 生成 30 条
    ...
]
```

### 指定分类生成

```python
generator = SparePartsDataGenerator()

# 只生成机械传动件
generator.generate_by_category("机械传动件", "A", 25)

# 保存数据
generator.save_to_json()
```

### 选择性检测

```python
checker = AIPartsChecker()

# 只检测命名和分类
check_items = ['naming', 'classification']

results = checker.check_batch(parts_data, check_items=check_items)
```

### 导入数据库

```python
import sqlite3
import json

# 加载数据
with open('data/spare_parts_data.json', 'r', encoding='utf-8') as f:
    parts = json.load(f)

# 连接数据库
conn = sqlite3.connect('app/spare_parts.db')
cursor = conn.cursor()

# 插入数据
for part in parts:
    cursor.execute("""
        INSERT INTO spare_parts 
        (part_code, part_name, spec_model, category_code, category_name,
         unit, unit_price, supplier, brand, material, weight, safety_stock)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
    """, (
        part['part_code'], part['part_name'], part['spec_model'],
        part['category_code'], part['category_name'], part['unit'],
        part['unit_price'], part['supplier'], part['brand'],
        part['material'], part['weight'], part['safety_stock']
    ))

conn.commit()
conn.close()
```

## ⚠️ 注意事项

### API 密钥安全
- ⚠️ **不要**将 `.env` 文件提交到 Git
- ✅ 在 `.gitignore` 中添加 `.env`
- ✅ 定期更换 API 密钥

### API 限流处理
- 脚本已自动添加延迟（5 秒/请求）
- 遇到 429 错误会自动重试
- 大批量生成时建议分批进行

### 数据质量
- AI 生成的数据需要人工审核
- 重点关注价格和供应商信息
- 定期更新和维护数据

### 性能优化
- 批量检测时使用异步处理
- 建立检测结果缓存
- 优化数据库索引

## 🐛 故障排查

### 问题 1：API 调用失败
```
错误：401 Unauthorized
原因：API 密钥无效或过期
解决：检查.env 文件中的 API 密钥是否正确
```

### 问题 2：JSON 解析失败
```
错误：JSON 解析失败
原因：AI 响应格式异常
解决：检查网络连接，重试生成
```

### 问题 3：数据生成过慢
```
现象：每类数据处理时间超过 1 分钟
原因：API 响应慢或网络延迟
解决：检查网络，避开高峰期
```

### 问题 4：AI 检测评分低
```
现象：大量数据评分低于 70 分
原因：生成提示词不够明确
解决：优化提示词，增加示例
```
