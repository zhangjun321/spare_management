# 库存状态定义说明

## 📊 库存状态概述

系统根据**当前库存**与**最低库存**、**最高库存**的对比关系，自动判断并更新备件的库存状态。

---

## 🎯 四种库存状态定义

### 1. ✅ 正常（normal）

**定义**：
- 当前库存 > 最低库存 **且** 当前库存 < 最高库存

**条件**：
```python
current_stock > min_stock AND (max_stock is None OR current_stock < max_stock)
```

**说明**：
- 库存量在合理范围内
- 不需要补货，也不需要处理
- 系统默认状态

**图标**：✓ 绿色对勾  
**颜色**：绿色（bg-success）

---

### 2. ⚠️ 低库存（low）

**定义**：
- 当前库存 ≤ 最低库存

**条件**：
```python
current_stock <= min_stock
```

**说明**：
- 库存已达到或低于最低警戒线
- 需要尽快安排补货
- 可能影响正常业务

**图标**：⚠️ 红色感叹号  
**颜色**：红色（bg-danger）

**示例**：
- 最低库存：10 个
- 当前库存：8 个
- 状态：🔴 低库存

---

### 3. 🔴 缺货（out）

**定义**：
- 当前库存 = 0

**条件**：
```python
current_stock == 0
```

**说明**：
- 库存已完全耗尽
- 急需补货
- 可能已影响业务

**注意**：缺货是低库存的极端情况，在列表中也会显示为"低库存"状态

**图标**：⚠️ 红色感叹号  
**颜色**：红色（bg-danger）

**示例**：
- 最低库存：10 个
- 当前库存：0 个
- 状态：🔴 缺货（严重低库存）

---

### 4. 🟡 超储/过剩（overstocked）

**定义**：
- 当前库存 ≥ 最高库存

**条件**：
```python
max_stock is not None AND current_stock >= max_stock
```

**说明**：
- 库存已超过最高警戒线
- 库存积压，占用资金
- 可能需要促销或调拨

**图标**：⚠️ 黄色感叹号  
**颜色**：黄色（bg-warning）

**示例**：
- 最高库存：100 个
- 当前库存：120 个
- 状态：🟡 超储

---

## 📈 状态判断逻辑流程图

```
开始
  ↓
当前库存 = 0?
  ↓ 是 → 缺货（红色）
  ↓ 否
当前库存 ≤ 最低库存？
  ↓ 是 → 低库存（红色）
  ↓ 否
当前库存 ≥ 最高库存？
  ↓ 是 → 超储（黄色）
  ↓ 否
正常（绿色）
```

## 🔧 自动更新机制

### 触发时机

系统在以下情况会自动更新库存状态：

1. **创建备件时**
   - 调用 `update_stock_status()` 方法
   - 自动计算并设置状态

2. **编辑备件时**
   - 修改库存数量后
   - 调用 `update_stock_status()` 方法

3. **入库/出库操作**
   - 批次管理、交易记录更新库存时
   - 自动触发状态更新

### 代码实现

```python
def update_stock_status(self):
    """更新库存状态"""
    if self.current_stock <= self.min_stock:
        self.stock_status = 'low'  # 低库存
    elif self.max_stock and self.current_stock >= self.max_stock:
        self.stock_status = 'overstocked'  # 超储
    else:
        self.stock_status = 'normal'  # 正常
```

## 📋 库存阈值设置建议

### 最低库存（min_stock）

**定义**：保证正常运营所需的最低库存量

**设置方法**：
```
最低库存 = 日均消耗量 × 采购提前期 + 安全库存
```

**示例**：
- 日均消耗：5 个/天
- 采购提前期：10 天
- 安全库存：20 个
- 最低库存 = 5 × 10 + 20 = 70 个

### 最高库存（max_stock）

**定义**：经济合理的最高库存量

**设置方法**：
```
最高库存 = 最低库存 + 经济订货批量
```

**考虑因素**：
- 仓库容量
- 资金占用
- 保质期限制
- 采购折扣

### 安全库存（safety_stock）

**定义**：应对突发需求的缓冲库存

**设置方法**：
```
安全库存 = (最大日均消耗 - 平均日均消耗) × 采购提前期
```

**说明**：
- 新增字段，2026-03-19 添加
- 用于更精细的库存管理
- 可与最低库存配合使用

**建议频率**：

**预警触发条件**：
- 库存 ≤ 最低库存

**通知方式**：

- 邮件通知
- 短信提醒

**定义**：触发采购建议的库存点


**应用**：

## ⚠️ 注意事项

### 1. 状态更新时机

- 库存变动后必须调用 `update_stock_status()`
- 批量导入数据时需要手动更新状态
- 修改 min_stock 或 max_stock 后需要重新计算

### 2. 边界值处理

- `current_stock = min_stock` → 低库存
- `current_stock = max_stock` → 超储
- `max_stock = NULL` → 不判断超储

### 3. 性能考虑

- 大批量数据更新时注意事务处理
- 避免频繁触发状态更新
- 可以考虑定时任务批量更新

## 📞 常见问题

### Q1: 为什么没有"缺货"状态选项？

**A**: 缺货是低库存的极端情况（current_stock = 0），系统统一归类为"低库存"状态，用红色标识。可以在筛选时选择"不足"来查看所有低库存和缺货的备件。

### Q2: 如何批量更新库存状态？

**A**: 可以创建 SQL 脚本或使用管理后台的批量操作功能：

```sql
UPDATE spare_part 
SET stock_status = CASE
    WHEN current_stock <= min_stock THEN 'low'
    WHEN max_stock IS NOT NULL AND current_stock >= max_stock THEN 'overstocked'
    ELSE 'normal'
END;
```

### Q3: 安全库存和最低库存有什么区别？

**A**: 
- **安全库存**：应对突发需求的缓冲，是理论计算值
- **最低库存**：实际业务中的补货触发点，可能包含安全库存
- 关系：最低库存 = 日均消耗 × 提前期 + 安全库存
