## 三、版本管理策略

### 3.1 分支管理模型

采用 **Git Flow** 工作流，适合企业级项目开发：

```
main (生产分支)
  │
  ├─── develop (开发分支)
  │      │
  │      ├─── feature/user-auth (功能分支)
  │      ├─── feature/inventory (功能分支)
  │      └─── feature/report (功能分支)
  │
  ├─── release/v1.0.0 (发布分支)
  │
  └─── hotfix/bug-fix (热修复分支)
```

#### 分支说明：

| 分支类型 | 命名规范 | 说明 | 保护级别 |
|---------|---------|------|---------|
| **主分支** | `main` | 生产环境代码，稳定版本 | 🔒 保护分支 |
| **开发分支** | `develop` | 日常开发集成分支 | 🔒 保护分支 |
| **功能分支** | `feature/xxx` | 新功能开发 | 开发者自行管理 |
| **发布分支** | `release/vX.Y.Z` | 版本发布准备 | 🔒 保护分支 |
| **热修复分支** | `hotfix/xxx` | 紧急 bug 修复 | 🔒 保护分支 |

### 3.2 版本号规范

采用 **语义化版本号（Semantic Versioning）**：`主版本号。次版本号.修订号`

- **主版本号（Major）**：不兼容的 API 修改
- **次版本号（Minor）**：向下兼容的功能性新增
- **修订号（Patch）**：向下兼容的问题修正

**示例**：
- `v1.0.0` - 第一个正式版本
- `v1.1.0` - 新增功能
- `v1.1.1` - Bug 修复

### 3.3 提交信息规范

采用 **Angular 提交规范**：

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### 提交类型（type）：

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(备件): 添加备件导入功能` |
| `fix` | Bug 修复 | `fix(库存): 修复库存计算错误` |
| `docs` | 文档更新 | `docs: 更新 README 文档` |
| `style` | 代码格式 | `style: 代码格式化` |
| `refactor` | 重构 | `refactor(用户): 重构用户认证模块` |
| `test` | 测试 | `test: 添加单元测试` |
| `chore` | 构建/工具 | `chore: 更新依赖版本` |

## 四、本地备份方案

### 4.1 本地 Git 仓库

**必须项**：每个开发环境都应有完整的本地 Git 仓库

```bash
# 初始化本地仓库
cd d:\Trae\spare_management
git init

# 配置用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 创建.gitignore
echo "__pycache__/" >> .gitignore
echo "*.pyc" >> .gitignore
echo ".env" >> .gitignore
echo "venv/" >> .gitignore
echo "*.db" >> .gitignore
echo ".DS_Store" >> .gitignore

# 首次提交
git add .
git commit -m "feat: 初始版本 - 备件管理系统 v1.0.0"
```

### 4.2 本地多副本备份

**推荐方案**：在不同物理位置保存多个副本

```
备份策略：
├── 主开发机：D:\Trae\spare_management（日常工作目录）
├── 外部硬盘：E:\Backup\spare_management（每日备份）
├── 公司服务器：\\server\backup\spare_management（每周备份）
└── 云存储：OneDrive/百度网盘（每月备份压缩包）
```

### 4.3 自动化备份脚本

创建 `backup.bat`（Windows）：

```batch
@echo off
chcp 65001
echo ========================================
echo 备件管理系统 - 自动备份脚本
echo ========================================
echo.

set BACKUP_DATE=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%
set BACKUP_DIR=E:\Backup\spare_management\%BACKUP_DATE%
set SOURCE_DIR=D:\Trae\spare_management

echo [1/4] 创建备份目录...
mkdir "%BACKUP_DIR%"

echo [2/4] 排除临时文件...
robocopy "%SOURCE_DIR%" "%BACKUP_DIR%" /MIR /XD __pycache__ venv .git /XF *.pyc *.db .env

echo [3/4] 压缩备份文件...
cd "%BACKUP_DIR%\.."
"C:\Program Files\7-Zip\7z.exe" a -tzip "spare_management_backup_%BACKUP_DATE%.zip" "%BACKUP_DATE%"

echo [4/4] 清理临时备份目录...
rmdir /s /q "%BACKUP_DIR%"

echo.
echo ========================================
echo 备份完成！
echo 备份文件：%BACKUP_DIR%\..\spare_management_backup_%BACKUP_DATE%.zip
echo ========================================
pause
```

## 五、GitCode 远程仓库配置

```bash
# 验证远程仓库
git remote -v

# 推送到远程
git branch -M main
git push -u origin main
```

## 六、日常开发工作流

### 6.1 开始新功能

```bash
# 1. 切换到开发分支
git checkout develop
git pull origin develop

# 2. 创建功能分支
git checkout -b feature/inventory-import

# 3. 开发功能...
# 编写代码 → 测试 → 提交
git add .
git commit -m "feat(库存): 添加 Excel 导入功能"

# 4. 推送到远程
git push -u origin feature/inventory-import
```

### 6.2 合并功能到开发分支

1. 在 GitCode 上创建 **Merge Request (MR)**
2. 选择源分支：`feature/inventory-import`
3. 选择目标分支：`develop`
4. 填写 MR 描述，说明功能变更
5. 等待代码审查（Code Review）
6. 审查通过后合并

### 6.3 发布新版本

```bash
# 1. 从 develop 创建发布分支
git checkout develop
git pull origin develop
git checkout -b release/v1.0.0

# 2. 版本测试和最终修复
# 测试 → 修复 → 提交

# 3. 更新版本号（在__init__.py 或版本文件）
# __version__ = "1.0.0"

git add .
git commit -m "chore: 发布版本 v1.0.0"

# 4. 合并到 main 和 develop
git checkout main
git merge release/v1.0.0
git tag -a v1.0.0 -m "发布版本 v1.0.0"

git checkout develop
git merge release/v1.0.0

# 5. 推送
git push origin main --tags
git push origin develop
git push origin release/v1.0.0
```

### 6.4 紧急 Bug 修复

```bash
# 1. 从 main 创建热修复分支
git checkout main
git pull origin main
git checkout -b hotfix/login-bug

# 2. 修复 bug
# 修复代码 → 测试 → 提交
git commit -m "fix(登录): 修复登录验证失败问题"

# 3. 合并到 main 和 develop
git checkout main
git merge hotfix/login-bug

git checkout develop
git merge hotfix/login-bug

# 4. 推送
git push origin main
git push origin develop
git push origin hotfix/login-bug
```

## 七、备份与恢复策略

### 7.1 备份频率

| 备份类型 | 频率 | 负责人 | 存储位置 |
|---------|------|--------|----------|
| **Git 提交** | 每日多次 | 所有开发者 | GitCode + 本地 |
| **本地备份** | 每日自动 | 自动化脚本 | 外部硬盘 |
| **完整备份** | 每周 | 系统管理员 | 公司服务器 |
| **归档备份** | 每月 | 系统管理员 | 云存储 |

### 7.2 灾难恢复流程

```
场景 1：本地代码丢失
  → 从 GitCode 克隆：git clone https://gitcode.net/username/repo.git

场景 2：GitCode 不可用
  → 从本地备份恢复：解压备份文件 → 继续使用

场景 3：所有远程仓库不可用
  → 从离线备份恢复 → 搭建临时 Git 服务器
```

### 7.3 恢复脚本

创建 `restore.bat`：

```batch
@echo off
chcp 65001
echo ========================================
echo 备件管理系统 - 恢复脚本
echo ========================================
echo.

set BACKUP_FILE=%1
set RESTORE_DIR=D:\Trae\spare_management

if "%BACKUP_FILE%"=="" (
    echo 用法：restore.bat <备份文件路径>
    pause
    exit /b
)

echo [1/3] 解压备份文件...
"C:\Program Files\7-Zip\7z.exe" x "%BACKUP_FILE%" -o"D:\Trae\temp_restore"

echo [2/3] 恢复文件...
robocopy "D:\Trae\temp_restore" "%RESTORE_DIR%" /MIR

echo [3/3] 清理临时文件...
rmdir /s /q "D:\Trae\temp_restore"

echo.
echo ========================================
echo 恢复完成！
echo ========================================
pause
```

## 十一、常见问题 FAQ

### Q1：GitCode 访问速度慢怎么办？
**A**：检查网络连接，使用 DNS 加速（如 114.114.114.114），或切换到华为云 CodeArts。

### Q2：如何保证代码不丢失？
**A**：遵循"3-2-1 备份原则"：3 份副本、2 种介质、1 份异地备份。

### Q3：多人协作时如何避免冲突？
**A**：使用功能分支开发，及时拉取最新代码，小步提交，频繁合并。

### Q4：如何回滚到之前的版本？
**A**：使用 `git log` 查看历史 → `git checkout <commit_id>` → `git revert` 或`git reset`。

### Q5：私有仓库收费吗？
**A**：GitCode 私有仓库免费，华为云 CodeArts 有免费额度，超出后按量计费。
