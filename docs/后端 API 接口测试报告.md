## 问题分析与解决方案

### 根本原因分析

#### 1. 登录接口 400 错误

**现象**: POST /login 返回 400 状态码

**可能原因**:
- CSRF token 验证失败（蓝图未豁免 CSRF）
- 表单字段名称不匹配
- 请求头 Content-Type 设置错误

**调查**:
查看 `app/routes/auth.py` 代码，登录路由逻辑正确，但可能存在 CSRF 保护问题。

**解决方案**:
```python
# 在 auth.py 中添加 CSRF 豁免
from app.extensions import csrf
csrf.exempt(auth_bp)
```

或在测试请求中添加 CSRF token。

#### 2. 仪表盘页面 404

**现象**: GET /dashboard 返回 404

**原因分析**:
- 蓝图注册路径为 `/`（见 `app/__init__.py` 第 130-131 行）
- 仪表盘路由定义为 `@dashboard_bp.route('/')` 
- 实际访问路径应该是根路径 `/` 而不是 `/dashboard`

**解决方案**:
测试脚本应该访问 `/` 而不是 `/dashboard`，或者修改蓝图注册：
```python
app.register_blueprint(dashboard_bp, url_prefix='/dashboard')
```

#### 3. 仓库 API 返回非 JSON

**现象**: GET /api/warehouses 返回内容无法解析为 JSON

**原因分析**:
- 路由需要登录认证（`@login_required` 装饰器）
- 未登录用户访问会被重定向到登录页面（返回 HTML）
- 测试脚本未先登录获取 session

**解决方案**:
在测试仓库 API 前，先执行登录获取 session：
```python
session = requests.Session()
session.post(f"{BASE_URL}/login", data={
    'username': 'admin',
    'password': 'admin123'
})
response = session.get(f"{BASE_URL}/api/warehouses")
```

#### 4. 备件代码检查失败

**现象**: POST /spare_parts/api/check-code 返回内容无法解析

**原因分析**:
- 同样是因为需要登录认证
- 未登录时返回登录页面 HTML

**解决方案**:
与仓库 API 相同，需要先登录。
