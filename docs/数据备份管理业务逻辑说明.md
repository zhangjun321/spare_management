# 数据备份管理功能 - 业务逻辑详细说明

## 一、功能概述

数据备份管理模块提供了完整的数据库备份和恢复解决方案，确保备件管理系统数据的安全性和可恢复性。

### 核心功能
1. **手动备份** - 用户可随时创建数据库备份
2. **备份恢复** - 从历史备份中恢复数据
3. **备份下载** - 下载备份文件到本地
4. **备份删除** - 删除不需要的备份
5. **自动清理** - 定期清理过期备份
6. **状态监控** - 实时监控备份状态和统计信息

## 二、业务流程

### 2.1 备份创建流程

```
用户发起备份请求
    ↓
创建备份记录（状态：pending）
    ↓
更新状态为 running
    ↓
解析数据库连接信息
    ↓
执行 mysqldump 命令
    ↓
压缩备份文件（GZIP）
    ↓
更新备份记录（状态：success/failed）
    ↓
返回备份结果
```

#### 详细步骤

**步骤 1: 创建备份记录**
```python
backup_record = DatabaseBackup(
    backup_name='backup_20260331_120000',
    backup_file='/path/to/backup.sql',
    backup_type='manual',  # 或 scheduled
    status='pending',
    created_by=user_id
)
```

**步骤 2: 解析数据库连接信息**
```python
from urllib.parse import urlparse

uri = urlparse('mysql://root:password@localhost:3306/spare_parts_db')
db_host = uri.hostname  # localhost
db_port = uri.port      # 3306
db_user = uri.username  # root
db_password = uri.password  # password
db_name = uri.path.lstrip('/')  # spare_parts_db
```

**步骤 3: 执行备份命令**
```bash
mysqldump -h localhost -P 3306 -u root -ppassword \
  --single-transaction \
  --routines \
  --triggers \
  spare_parts_db > backup.sql
```

**步骤 4: 压缩备份文件**
```python
import gzip
import shutil

with open('backup.sql', 'rb') as f_in:
    with gzip.open('backup.sql.gz', 'wb') as f_out:
        shutil.copyfileobj(f_in, f_out)
```

**步骤 5: 更新备份记录**
```python
backup_record.status = 'success'
backup_record.backup_size = os.path.getsize('backup.sql.gz')
backup_record.end_time = datetime.now()
backup_record.remark = f'备份成功，文件大小：{size_mb} MB'
```

### 2.2 备份恢复流程

```
用户选择备份并确认恢复
    ↓
验证备份记录存在
    ↓
验证备份文件存在
    ↓
解析数据库连接信息
    ↓
解压并执行恢复命令
    ↓
验证恢复结果
    ↓
返回恢复结果
```

#### 详细步骤

**步骤 1: 验证备份**
```python
backup = DatabaseBackup.query.get(backup_id)
if not backup:
    return False, '备份记录不存在'

if not os.path.exists(backup.backup_file):
    return False, '备份文件不存在'
```

**步骤 2: 执行恢复命令**
```bash
# 如果是 GZIP 压缩文件
gunzip -c backup.sql.gz | mysql -h localhost -P 3306 -u root -ppassword spare_parts_db

# 如果是普通 SQL 文件
mysql -h localhost -P 3306 -u root -ppassword spare_parts_db < backup.sql
```

**步骤 3: 处理恢复结果**
```python
result = subprocess.run(cmd, shell=True, capture_output=True, text=True)

if result.returncode != 0:
    raise Exception(f'恢复失败：{result.stderr}')
else:
    return True, '恢复成功'
```

## 三、数据模型

### DatabaseBackup 表结构

| 字段名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| id | Integer | 主键 ID | 自增 |
| backup_name | String(200) | 备份名称 | - |
| backup_file | String(500) | 备份文件路径 | - |
| backup_size | BigInteger | 文件大小（字节） | - |
| backup_type | String(20) | 备份类型 | manual |
| status | String(20) | 状态 | pending |
| start_time | DateTime | 开始时间 | - |
| end_time | DateTime | 结束时间 | - |
| remark | Text | 备注信息 | - |
| created_by | Integer | 创建人 ID | - |
| created_at | DateTime | 创建时间 | now() |

### 备份状态说明

| 状态 | 说明 | 颜色标识 |
|------|------|----------|
| pending | 等待执行 | 黄色 |
| running | 正在执行 | 蓝色 |
| success | 执行成功 | 绿色 |
| failed | 执行失败 | 红色 |

## 四、技术实现

### 4.1 备份目录管理

```python
BACKUP_DIR = os.path.join(
    os.path.dirname(os.path.dirname(__file__)), 
    'backups'
)

def init_backup_dir():
    if not os.path.exists(BACKUP_DIR):
        os.makedirs(BACKUP_DIR)
```

### 4.2 备份命名规则

```
backup_YYYYMMDD_HHMMSS.sql.gz
示例：backup_20260331_143022.sql.gz
```

### 4.3 错误处理

```python
try:
    # 执行备份操作
    backup_record.status = 'running'
    db.session.commit()
    
    # 执行 mysqldump
    result = subprocess.run(cmd, stdout=f, stderr=subprocess.PIPE)
    
    if result.returncode != 0:
        raise Exception(f'mysqldump 失败：{result.stderr.decode("utf-8")}')
    
    # 成功
    backup_record.status = 'success'
    
except Exception as e:
    # 失败处理
    backup_record.status = 'failed'
```

## 六、安全考虑

### 6.1 权限控制
- 创建备份：需要 `system.write` 权限
- 恢复备份：需要 `system.admin` 权限
- 删除备份：需要 `system.write` 权限
- 下载备份：需要 `system.read` 权限

### 6.2 数据保护
- 备份文件存储在服务器本地，不对外公开
- 恢复操作需要二次确认
- 记录所有备份操作的日志

### 6.3 密码安全
- 数据库密码从配置文件读取
- 命令行参数中使用 `-p` 传递密码（不在日志中显示）

## 七、性能优化

### 7.1 压缩优化
- 使用 GZIP 压缩，通常可减少 70-90% 的文件大小
- 压缩过程流式处理，不占用大量内存

### 7.2 备份策略建议
- 小规模数据库（<1GB）：每天备份
- 中等规模数据库（1-10GB）：每 6 小时备份
- 大规模数据库（>10GB）：使用增量备份

### 7.3 清理策略
- 保留最近 7 天的每日备份
- 保留最近 4 周的每周备份
- 保留最近 12 个月的每月备份

## 九、故障排查

### 9.1 备份失败常见原因

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| mysqldump 命令不存在 | 未安装 MySQL 客户端 | 安装 MySQL 或 MySQL Client |
| 权限拒绝 | 数据库用户无权限 | 授予 SELECT, SHOW VIEW 权限 |
| 磁盘空间不足 | 服务器磁盘已满 | 清理磁盘空间 |
| 连接超时 | 数据库连接失败 | 检查数据库服务状态 |

### 9.2 恢复失败常见原因

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 备份文件不存在 | 文件被删除 | 选择其他备份 |
| SQL 语法错误 | 版本不兼容 | 检查 MySQL 版本 |
| 外键约束冲突 | 数据依赖问题 | 使用 --disable-keys 选项 |
