## 🔧 修复方案

### 修改 MainLayout.jsx

**关键改动**:

```javascript
// 修改前（有问题）
<div 
  style={{
    marginLeft: sidebarCollapsed ? '64px' : '240px',
    width: 'calc(100% - ' + (sidebarCollapsed ? '64px' : '240px') + ')'  // ❌ 冲突
  }}
>
  <main style={{ width: '100%' }}>
    <Container fluid className="px-4">  // ❌ 额外的 padding
      {children}
    </Container>
  </main>
</div>

// 修改后（正确）
<div 
  style={{
    marginLeft: sidebarCollapsed ? '64px' : '240px',
    minWidth: 0,              // ✅ 允许缩小
    flex: '1 1 auto'          // ✅ 正确的 flex 布局
  }}
>
  <main style={{ width: '100%', overflowX: 'auto' }}>
    <Container fluid className="px-0" style={{ 
      maxWidth: '100%', 
      width: '100%', 
      margin: 0, 
      padding: 0 
    }}>
      {children}
    </Container>
  </main>
</div>
```

**技术要点**:
1. ✅ 移除冲突的 `width: calc()`
2. ✅ 使用 `flex: '1 1 auto'` 自动填充可用空间
3. ✅ 添加 `minWidth: 0` 允许 flex 项目缩小
4. ✅ 移除 Container 的额外 padding（`px-4` → `px-0`）
5. ✅ 设置 `margin: 0, padding: 0` 确保占满全宽

## 📊 技术细节

### Flexbox 布局原理

```css
/* 外层容器 */
.d-flex {
  display: flex;
}

/* 内容区域 */
.flex-grow-1 {
  flex-grow: 1;  /* 占据所有可用空间 */
}

/* 自定义样式 */
{
  minWidth: 0;      /* 允许缩小到内容以下 */
  flex: 1 1 auto;   /* 可以增长、可以缩小、基准为 auto */
}
```

### 为什么使用 `minWidth: 0`

默认情况下，flex 项目的 `min-width` 为 `auto`，这意味着：
- 它不会缩小到内容以下
- 可能导致溢出或布局问题

设置 `minWidth: 0` 后：
- ✅ 允许 flex 项目缩小到内容以下
- ✅ 正确填充可用空间
- ✅ 避免溢出

### Container 组件的 padding

**Bootstrap Container 默认行为**:
```css
.container-fluid {
  padding-left: 15px;
  padding-right: 15px;
}
```

**移除 padding**:
```javascript
<Container fluid className="px-0">  // px-0 = padding-left: 0, padding-right: 0
```
