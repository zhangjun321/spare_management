## 问题清单

### 问题 1: User 模型缺少 managed_warehouses 关系

**错误信息**:
```
sqlalchemy.exc.InvalidRequestError: Mapper 'Mapper[User(user)]' has no property 'managed_warehouses'
```

**问题原因**:
- `Warehouse` 模型定义了 `manager` 关系，使用 `back_populates='managed_warehouses'`
- 但 `User` 模型中没有定义对应的 `managed_warehouses` 关系
- SQLAlchemy 无法自动建立双向关系

**修复方案**:
在 `User` 模型中添加 `managed_warehouses` 关系

**修复代码**:
```python
# app/models/user.py
class User(db.Model, UserMixin):
    # ... 其他字段 ...
    
    # 关系
    department = db.relationship('Department', foreign_keys=[department_id])
    role = db.relationship('Role', foreign_keys=[role_id], backref='role_users')
    managed_warehouses = db.relationship('Warehouse', foreign_keys='Warehouse.manager_id', back_populates='manager')
```

**修复结果**: ✅ 成功修复

### 问题 2: WarehouseZone 模型缺少 shelves 属性

**错误信息**:
```
sqlalchemy.exc.InvalidRequestError: Mapper 'Mapper[WarehouseZone(warehouse_zone)]' has no property 'shelves'
```

**问题原因**:
- `WarehouseLocation` 模型定义了 `zone` 关系，使用 `back_populates='shelves'`
- 但 `WarehouseZone` 模型中没有定义 `shelves` 关系
- 实际上项目中没有 `WarehouseShelf` 模型，而是使用 `WarehouseRack`

**修复方案**:
修改 `WarehouseLocation` 模型中的关系配置，移除 `back_populates`，使用 `foreign_keys` 替代

**修复代码**:
```python
# app/models/warehouse_location.py
class WarehouseLocation(db.Model):
    # ... 其他字段 ...
    
    # 关系
    warehouse = db.relationship('Warehouse', back_populates='locations')
    zone = db.relationship('WarehouseZone', foreign_keys=[zone_id])  # 修改：移除 back_populates
    shelf = db.relationship('WarehouseRack', foreign_keys=[shelf_id])  # 修改：WarehouseShelf -> WarehouseRack
    batches = db.relationship('Batch', foreign_keys='Batch.location_id', back_populates='location', lazy='dynamic')
    spare_parts = db.relationship('SparePart', foreign_keys='SparePart.location_id', back_populates='warehouse_location', lazy='dynamic')
    inventory_records = db.relationship('InventoryRecord', foreign_keys='InventoryRecord.location_id', back_populates='location', lazy='dynamic')
```

**修复结果**: ✅ 成功修复

### 问题 3: WarehouseLocation 外键引用错误

**错误信息**:
```
sqlalchemy.exc.NoForeignKeysError: Could not determine join condition between parent/child tables on relationship WarehouseLocation.shelf - there are no foreign keys linking these tables
```

**问题原因**:
- `WarehouseLocation.shelf_id` 的外键引用了不存在的表 `warehouse_shelf`
- 实际应该引用 `warehouse_rack` 表
- 表名不匹配导致 SQLAlchemy 无法建立关系

**修复方案**:
修改外键引用，将 `warehouse_shelf.id` 改为 `warehouse_rack.id`

**修复代码**:
```python
# app/models/warehouse_location.py
class WarehouseLocation(db.Model):
    # ... 其他字段 ...
    
    # 修改前:
    shelf_id = db.Column(db.Integer, db.ForeignKey('warehouse_shelf.id'), nullable=True, index=True, comment='货架 ID')
    
    # 修改后:
    shelf_id = db.Column(db.Integer, db.ForeignKey('warehouse_rack.id'), nullable=True, index=True, comment='货架 ID')
```

**修复结果**: ✅ 成功修复

## 技术说明

### SQLAlchemy 关系配置最佳实践

1. **双向关系** - 使用 `back_populates`:
   ```python
   # 父模型
   children = db.relationship('Child', back_populates='parent')
   
   # 子模型
   parent = db.relationship('Parent', back_populates='children')
   ```

2. **单向关系** - 使用 `foreign_keys`:
   ```python
   # 只需要在一个方向访问关系时
   parent = db.relationship('Parent', foreign_keys=[parent_id])
   ```

3. **避免循环依赖**:
   - 如果关系配置复杂，优先使用 `foreign_keys`
   - 避免过多的 `back_populates` 导致初始化问题

### 本次修复的技术要点

1. **User.managed_warehouses**
   - 使用 `foreign_keys='Warehouse.manager_id'` 明确指定外键
   - 使用 `back_populates='manager'` 建立双向关系

2. **WarehouseLocation.zone**
   - 移除 `back_populates`，因为 `WarehouseZone` 不需要访问所有位置
   - 使用 `foreign_keys=[zone_id]` 建立单向关系

3. **WarehouseLocation.shelf**
   - 修正模型名称：`WarehouseShelf` → `WarehouseRack`
   - 使用 `foreign_keys=[shelf_id]` 建立单向关系
