# 环境配置指南

## 快速开始

### 1. 复制配置文件

```bash
# 在项目根目录执行
cp .env.example .env
```

### 2. 配置 API 密钥

编辑 `.env` 文件，填入你的实际 API 密钥：

#### 硅基流动 AI（必需）
```env
SILICONFLOW_API_KEY=your_actual_api_key_here
```

#### 百度千帆 AI（用于 AI 推荐功能，强烈推荐）
```env
QIANFAN_API_KEY=your_baidu_api_key_here
QIANFAN_SECRET_KEY=your_baidu_secret_key_here
```

**获取方式**:
1. 访问 [百度千帆控制台](https://console.bce.baidu.com/qianfan/)
2. 创建应用获取 API Key 和 Secret Key
3. 免费额度：新用户有一定免费调用次数

#### 百度图像生成（可选）
```env
BAIDU_IMAGE_API_KEY=your_baidu_image_api_key_here
BAIDU_IMAGE_SECRET_KEY=your_baidu_image_secret_key_here
```

### 3. 配置数据库

**SQLite（默认，适合开发）**:
```env
DATABASE_URL=sqlite:///app/spare_parts.db
```

**MySQL（生产环境推荐）**:
```env
DATABASE_URL=mysql://username:password@localhost:3306/spare_management
```

### 4. 安全配置

**开发环境**:
```env
FLASK_ENV=development
FLASK_DEBUG=True
SECRET_KEY=dev-secret-key-change-in-production
```

**生产环境**:
```env
FLASK_ENV=production
FLASK_DEBUG=False
SECRET_KEY=<使用强随机密钥>
WTF_CSRF_ENABLED=True
```

生成随机密钥：
```python
import secrets
print(secrets.token_hex(32))
```

## 配置项说明

### AI 服务配置

| 配置项 | 说明 | 必需 | 默认值 |
|--------|------|------|--------|
| `SILICONFLOW_API_KEY` | 硅基流动 API 密钥 | 是 | - |
| `SILICONFLOW_API_URL` | 硅基流动 API 地址 | 否 | `https://api.siliconflow.cn/v1` |
| `SILICONFLOW_MODEL` | 硅基流动模型 | 否 | `Qwen/Qwen2.5-72B-Instruct` |
| `QIANFAN_API_KEY` | 百度千帆 API 密钥 | 推荐 | - |
| `QIANFAN_SECRET_KEY` | 百度千帆 Secret 密钥 | 推荐 | - |
| `QIANFAN_MODEL` | 百度千帆模型 | 否 | `ERNIE-Bot-4.0` |
| `BAIDU_IMAGE_API_KEY` | 百度图像 API 密钥 | 否 | - |
| `BAIDU_IMAGE_SECRET_KEY` | 百度图像 Secret 密钥 | 否 | - |

### 数据库配置

| 配置项 | 说明 | 必需 | 默认值 |
|--------|------|------|--------|
| `DATABASE_URL` | 数据库连接字符串 | 是 | `sqlite:///app/spare_parts.db` |

**数据库连接字符串格式**:
- SQLite: `sqlite:///path/to/database.db`
- MySQL: `mysql://user:password@host:port/database`
- PostgreSQL: `postgresql://user:password@host:port/database`

### 应用配置

| 配置项 | 说明 | 开发环境 | 生产环境 |
|--------|------|----------|----------|
| `FLASK_ENV` | Flask 环境 | `development` | `production` |
| `FLASK_DEBUG` | 调试模式 | `True` | `False` |
| `SECRET_KEY` | 密钥 | 自定义 | 强随机密钥 |
| `WTF_CSRF_ENABLED` | CSRF 保护 | `True` | `True` |

### 文件上传配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `UPLOAD_FOLDER` | 上传文件夹 | `app/static/uploads` |
| `MAX_CONTENT_LENGTH` | 最大文件大小 (bytes) | `16777216` (16MB) |
| `ALLOWED_EXTENSIONS` | 允许的扩展名 | `png,jpg,jpeg,gif,webp` |

4. 复制到 `.env` 文件

## 常见问题

### Q1: 配置后应用无法启动

**检查步骤**:
1. 确认 `.env` 文件在正确位置
2. 检查 API 密钥格式是否正确
3. 查看错误日志

```bash
python run.py 2>&1 | head -50
```

### Q2: AI 功能无法使用

**可能原因**:
- API 密钥无效或过期
- 网络问题无法访问 AI 服务
- 免费额度已用完

**解决方法**:
```bash
# 测试 API 连接
python -c "from app.services.baidu_qianfan_service import BaiduQianfanService; s = BaiduQianfanService(); print(s.chat([{'role': 'user', 'content': '你好'}]))"
```

### Q3: CSRF 错误

**开发环境临时禁用 CSRF**:
```python
# 在 app/__init__.py 中
if app.config['DEBUG']:
    csrf.exempt(some_blueprint)
```

**生产环境**:
- 确保 `WTF_CSRF_ENABLED=True`
- 前端正确获取和发送 CSRF token

### Q4: 数据库连接失败

**SQLite**:
- 确保数据库文件路径存在
- 检查文件权限

**MySQL**:
```bash
# 测试连接
mysql -u username -p -h localhost spare_management
```

## 最佳实践

```env
FLASK_ENV=development
FLASK_DEBUG=True
SECRET_KEY=dev-key-not-for-production
WTF_CSRF_ENABLED=False  # 开发方便，生产必须启用
```

### 生产环境

```env
FLASK_ENV=production
FLASK_DEBUG=False
SECRET_KEY=<64 位随机字符串>
WTF_CSRF_ENABLED=True
WTF_CSRF_TIME_LIMIT=3600
DATABASE_URL=mysql://user:pass@host:3306/db
DATABASE_POOL_SIZE=10
DATABASE_MAX_OVERFLOW=20
```

### Docker 环境

使用环境变量：
```bash
docker run -e SILICONFLOW_API_KEY=xxx -e QIANFAN_API_KEY=xxx your-image
```

或使用 Docker Compose：
```yaml
environment:
  - SILICONFLOW_API_KEY=${SILICONFLOW_API_KEY}
  - QIANFAN_API_KEY=${QIANFAN_API_KEY}
  - DATABASE_URL=mysql://mysql:3306/spare_management
```

## 安全提醒

⚠️ **重要安全注意事项**:

1. **不要提交 `.env` 文件到 Git**
   ```bash
   # .env 已在 .gitignore 中
   echo ".env" >> .gitignore
   ```

2. **生产环境使用强密钥**
   ```python
   import secrets
   SECRET_KEY = secrets.token_hex(32)
   ```

3. **定期更新 API 密钥**
   - 建议每 3-6 个月更新一次
   - 发现泄露立即更新

4. **限制 API 权限**
   - 使用最小权限原则
   - 为不同服务创建不同的 API Key

## 故障排查

### 日志查看

```bash
# 查看应用日志
tail -f logs/app.log

# 查看错误日志
tail -f logs/error.log
```

### 配置检查脚本

```python
# scripts/check_config.py
import os
from dotenv import load_dotenv

load_dotenv()

required_vars = [
    'SILICONFLOW_API_KEY',
    'DATABASE_URL'
]

missing = [var for var in required_vars if not os.getenv(var)]
if missing:
    print(f"❌ 缺少配置：{missing}")
else:
    print("✅ 配置检查通过")
```

运行检查：
```bash
python scripts/check_config.py
```
