## 🎯 一、设计背景与目标

### 1.1 业务需求分析

基于项目现有仓库管理模块的功能，结合备件管理行业特点，进行深度功能升级：

**现有功能：**
- ✅ 仓库基础信息管理
- ✅ 库位管理
- ✅ 库存概览
- ✅ 库存调拨、调整
- ✅ 基础报表

**升级目标：**
- 🎯 智能化：引入AI辅助决策
- 🎯 精细化：三级库位管理
- 🎯 可视化：3D/2D仓库可视化
- 🎯 协同化：与采购、设备、维修模块深度集成
- 🎯 绿色化：能耗优化与可持续运营

---

### 1.2 行业趋势分析（2025-2026）

基于最新行业研究，现代仓储管理呈现以下核心趋势：

#### 趋势1：AI与自动化深度融合
- **智能调度**：AI算法优化任务分配和路径规划
- **计算机视觉**：自动识别货物状态、破损检测
- **预测性分析**：需求预测、库存预警
- **拣选效率**：提升35%，错误率降至0.2%以下

#### 趋势2：软件定义仓储
- **统一调度平台**：跨品牌、跨类型自动化设备协同
- **软硬件解耦**：灵活适配不同硬件方案
- **开放生态**：API驱动的系统集成

#### 趋势3：数字孪生与虚拟仿真
- **3D仓库可视化**：实时状态监控
- **策略验证**：虚拟空间验证仓储策略，试错成本降低92%
- **跨场景适配**：知识迁移，缩短研发周期80%

#### 趋势4：低碳化与可持续发展
- **绿色算法**：优化照明、温控能耗
- **路径优化**：最短拣货路径，降低能耗
- **环保包装**：智能推荐包装方案

#### 趋势5：系统边界消融
- **向上集成**：与MES、APS系统对接
- **向下延伸**：自动化设备统一调度
- **供应链协同**：从仓储模块到协同平台

## 📦 三、核心功能详细设计

### 3.1 三级库位管理系统

#### 功能描述
实现库区（Zone）→ 货架（Rack）→ 货位（Location）三级精细化管理

#### 数据模型设计

**WarehouseZone（库区表）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| warehouse_id | Integer | 仓库ID（外键） |
| zone_code | String(20) | 库区编码（唯一） |
| zone_name | String(100) | 库区名称 |
| zone_type | String(20) | 库区类型：general/cold/hazardous/elevated |
| temperature_min | Numeric(5,2) | 最低温度要求 |
| temperature_max | Numeric(5,2) | 最高温度要求 |
| humidity_min | Numeric(5,2) | 最低湿度要求 |
| humidity_max | Numeric(5,2) | 最高湿度要求 |
| description | Text | 描述 |
| sort_order | Integer | 排序 |
| is_active | Boolean | 是否启用 |

**WarehouseRack（货架表）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| zone_id | Integer | 库区ID（外键） |
| rack_code | String(20) | 货架编码（唯一） |
| rack_name | String(100) | 货架名称 |
| rack_type | String(20) | 货架类型：standard/cantilever/drive_in |
| levels_count | Integer | 层数（默认5） |
| max_weight_per_level | Numeric(10,2) | 每层最大承重(kg) |
| position_x | Integer | X坐标（用于可视化） |
| position_y | Integer | Y坐标（用于可视化） |
| sort_order | Integer | 排序 |
| is_active | Boolean | 是否启用 |

**WarehouseLocation（货位表，升级）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| rack_id | Integer | 货架ID（外键，新增） |
| warehouse_id | Integer | 仓库ID（外键） |
| location_code | String(50) | 货位编码（唯一） |
| location_name | String(100) | 货位名称 |
| location_type | String(20) | 货位类型 |
| level | Integer | 层号 |
| position | Integer | 位号 |
| max_capacity | Integer | 最大容量(件) |
| max_weight | Numeric(10,2) | 最大承重(kg) |
| current_capacity | Integer | 当前容量 |
| abc_class | String(1) | ABC分类：A/B/C（新增） |
| is_quick_pick | Boolean | 是否快拣位（新增） |
| is_reserved | Boolean | 是否保留（新增） |
| status | String(20) | 状态：available/occupied/locked/maintenance |

#### 核心功能
1. **库位自动编码**：按规则自动生成库区-货架-货位编码
2. **容量智能校验**：入库时自动校验货位容量和承重
3. **ABC分类分配**：根据备件价值自动推荐货位
4. **快拣位管理**：高频备件自动分配至快拣区
5. **温湿度监控**：特殊库区温湿度实时监控（集成物联网设备）

**InventoryCheck（盘点单表）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| check_code | String(50) | 盘点单号（唯一） |
| check_name | String(100) | 盘点名称 |
| check_type | String(20) | 盘点类型：full/cyclical/abc/random |
| warehouse_id | Integer | 仓库ID（外键） |
| location_id | Integer | 库位ID（外键，可选） |
| category_id | Integer | 分类ID（外键，可选） |
| abc_class | String(1) | ABC分类（可选） |
| start_date | DateTime | 开始日期 |
| end_date | DateTime | 结束日期 |
| status | String(20) | 状态：draft/in_progress/completed/cancelled |
| total_items | Integer | 总盘点项数 |
| checked_items | Integer | 已盘点项数 |
| discrepancy_items | Integer | 差异项数 |
| discrepancy_value | Numeric(15,2) | 差异金额 |
| checked_by | Integer | 盘点人ID（外键） |
| approved_by | Integer | 审核人ID（外键） |
| approved_at | DateTime | 审核时间 |

**InventoryCheckItem（盘点明细表）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| inventory_check_id | Integer | 盘点单ID（外键） |
| spare_part_id | Integer | 备件ID（外键） |
| batch_id | Integer | 批次ID（外键，可选） |
| warehouse_id | Integer | 仓库ID（外键） |
| location_id | Integer | 库位ID（外键，可选） |
| system_quantity | Integer | 系统数量 |
| check_quantity | Integer | 盘点数量 |
| second_check_quantity | Integer | 复盘数量 |
| difference_quantity | Integer | 差异数量 |
| difference_value | Numeric(15,2) | 差异金额 |
| status | String(20) | 状态：pending/checked/difference/resolved |
| is_resolved | Boolean | 是否已处理 |
| checked_by | Integer | 盘点人ID（外键） |
| checked_at | DateTime | 盘点时间 |
| second_checked_by | Integer | 复盘人ID（外键） |
| second_checked_at | DateTime | 复盘时间 |

#### 盘点类型说明

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 全盘（full） | 全仓库所有物料全面盘点 | 年终/季度大盘点 |
| 循环盘点（cyclical） | 按周期轮流盘点，上次盘点超过30天的物料 | 日常盘点，减少停产 |
| ABC分类盘点（abc） | 按ABC分类分别盘点，A类高频 | 重点物料精准管理 |
| 随机抽盘（random） | 随机抽取20%物料盘点 | 快速抽查，验证准确性 |

#### 核心功能
1. **智能盘点范围选择**：根据条件自动生成盘点清单
2. **移动端盘点**：支持PDA/手机APP扫码盘点
3. **复盘机制**：差异项自动触发复盘流程
4. **差异处理**：支持差异调整、审核流程
5. **盘点报告**：自动生成盘点差异报告
6. **库存准确率计算**：基于盘点结果计算准确率KPI

### 3.3 库龄分析与呆滞库存管理

#### 功能描述
实时监控库存库龄，识别呆滞、过时库存，及时预警

#### 数据模型设计

**StockAgeAnalysis（库龄分析表）**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| warehouse_id | Integer | 仓库ID（外键） |
| spare_part_id | Integer | 备件ID（外键） |
| batch_id | Integer | 批次ID（外键，可选） |
| stock_age_days | Integer | 库龄天数 |
| stock_age_level | String(20) | 库龄等级：normal/warning/danger/critical |
| quantity | Integer | 库存数量 |
| unit_price | Numeric(15,2) | 单价 |
| total_value | Numeric(15,2) | 库存价值 |
| is_slow_moving | Boolean | 是否呆滞（&gt;90天） |
| is_obsolete | Boolean | 是否过时（&gt;365天） |
| last_outbound_date | DateTime | 最后出库时间 |
| days_since_last_outbound | Integer | 距最后出库天数 |
| analysis_date | DateTime | 分析日期 |

#### 库龄分级标准

| 等级 | 库龄天数 | 说明 | 预警级别 |
|------|----------|------|----------|
| 正常（normal） | ≤30天 | 健康库存 | - |
| 警告（warning） | 31-90天 | 需要关注 | ⚠️ 黄色 |
| 危险（danger） | 91-180天 | 呆滞风险 | 🟠 橙色 |
| 严重（critical） | &gt;180天 | 严重呆滞 | 🔴 红色 |

#### 核心功能
1. **自动库龄计算**：每日自动计算所有库存的库龄
2. **呆滞库存识别**：自动标记90天以上未动库存
3. **过时库存标记**：超过365天标记为过时
4. **最后出库追踪**：记录每个批次最后出库时间
5. **预警通知**：库龄超标自动触发通知
6. **处理建议**：AI自动给出处理建议（促销/调拨/报废）

### 3.4 ABC分类管理系统

#### 功能描述
基于帕累托法则（80/20法则）对备件进行价值分类，优化管理资源

#### 分类标准

| 分类 | 价值占比 | 物料占比 | 管理策略 | 盘点频率 |
|------|----------|----------|----------|----------|
| A类 | 前80% | 约10-20% | 重点管理，精细控制 | 月度/季度 |
| B类 | 80-95% | 约20-30% | 常规管理 | 季度/半年 |
| C类 | 95%+ | 约50-70% | 简化管理 | 年度 |

#### 核心功能
1. **自动分类计算**：基于库存价值自动计算ABC分类
2. **分类可视化**：ABC分类饼图、分布图
3. **差异化管理策略**：
   - A类：高频率盘点、安全库存严格控制
   - B类：标准管理流程
   - C类：简化流程、批量管理
4. **分类调整**：支持人工调整分类
5. **分类趋势分析**：分类变化趋势追踪

### 3.7 KPI指标体系

#### 仓库运营KPI

| KPI指标 | 计算公式 | 目标值 | 说明 |
|---------|----------|--------|------|
| **库位利用率** | 已用库位数/总库位数 × 100% | 70-85% | 过高拥堵，过低浪费 |
| **库存周转率** | 出库成本/平均库存价值 | ≥4次/年 | 越高越好 |
| **订单履约率** | 按时完成订单数/总订单数 × 100% | ≥98% | 服务水平指标 |
| **库存准确率** | (总项数-差异项数)/总项数 × 100% | ≥99.5% | 盘点准确率 |
| **拣货效率** | 拣货行数/工时 | 持续提升 | 作业效率 |
| **入库及时率** | 按时入库单数/总入库单数 × 100% | ≥95% | 收货效率 |
| **呆滞库存占比** | 呆滞库存价值/总库存价值 × 100% | ≤5% | 库存健康度 |
| **盘点差异率** | 差异项数/总盘点项数 × 100% | ≤0.5% | 库存准确性 |

#### KPI仪表盘
- **实时KPI卡片**：核心指标一目了然
- **趋势图表**：KPI变化趋势
- **对比分析**：多仓库、多期间对比
- **目标追踪**：实际值vs目标值
